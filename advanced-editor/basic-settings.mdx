### Enhanced Caller ID Management

Configure and manage client phone numbers for outbound calling with advanced verification and status tracking:

<Tabs>
  <Tab title="Adding Caller IDs">
    <Steps>
      <Step title="Initiate Addition">
        Click "Add Caller ID" to open the configuration dialog
      </Step>
      <Step title="Enter Phone Information">
        Provide a descriptive name and the phone number for verification
      </Step>
      <Step title="Automatic Verification Call">
        Twilio initiates a verification call within 30 seconds
      </Step>
      <Step title="Code Entry">
        Enter the verification code provided during the automated call
      </Step>
      <Step title="Status Confirmation">
        Verify the caller ID status shows as "Verified" and ready for use
      </Step>
    </Steps>

    <Info>
    **Automatic Process**: Twilio automatically calls the provided number with a verification code - no manual intervention required.
    </Info>
  </Tab>
  <Tab title="Advanced Management">
    **Comprehensive Status Tracking:**
    - **✅ Verified**: Ready for outbound calls
    - **⏳ Pending**: Awaiting verification completion
    - **❌ Failed**: Verification failed, requires retry
    - **🔄 Processing**: Verification in progress

    **Management Actions:**
    - **Check Status**: Real-time validation with Twilio
    - **Update Status**: Refresh verification state
    - **Retry Verification**: Restart failed verification process
    - **Delete**: Remove caller ID from agent configuration
  </Tab>
  <Tab title="Verification Troubleshooting">
    <Warning>
    **Code Timing**: Verification codes are time-limited. Complete the process promptly after receiving the automated call.
    </Warning>

    **Common Solutions:**
    - Ensure the phone number can receive automated calls
    - Check for carrier restrictions on robotic calls
    - Verify number format is correct (+1XXXXXXXXXX for US)
    - Use "Retry Verification" for failed attempts
    - Allow up to 2 minutes for verification call delivery
  </Tab>
</Tabs>

## Advanced Conversational Behavior Configuration

### Response Management

Fine-tune how your agent handles conversation timing and responses:

<Tabs>
  <Tab title="Response Settings">
    **Eagerness Control:**
    - **Low**: More patient, allows longer pauses before responding
    - **Normal**: Balanced response timing for natural conversation flow (recommended)
    - **High**: Quick responses, minimal wait time
    - **Auto**: AI-determined optimal timing based on context

    **Timeout Configuration:**
    - **Turn Timeout**: Time to wait before considering caller finished speaking (6 seconds default)
    - **Silence End Call Timeout**: Maximum silence before ending call (25 seconds default)
    - **Spelling Patience**: Handle spelling requests (auto/enabled/disabled)

    <Info>
    **Default Optimization**: Standard settings (Normal eagerness, 6s turn timeout, 25s silence timeout) are optimized for professional phone conversations.
    </Info>
  </Tab>
  <Tab title="Call Duration Management">
    **Duration Controls:**
    - **Maximum Duration**: Automatic call termination limit (600 seconds default)
    - **End Call Message**: Custom message when duration limit is reached
    - **Speculative Turn**: Enable predictive response preparation
    - **Background Speech Filter**: Filter out background noise and conversations

    **Advanced Features:**
    - Real-time conversation monitoring
    - Automatic quality adjustments
    - Dynamic timeout adaptation
    - Context-aware behavior modification

    <Warning>
    **Duration Limits**: Set appropriate maximum call durations to manage costs while allowing sufficient time for complex conversations.
    </Warning>
  </Tab>
