## Overview

The Actions & Tools section is the powerhouse of your AI agent's operational capabilities. Here you configure what your agent can do during and after calls - from transferring conversations to human agents or SIP systems, to making intelligent API calls, and setting up sophisticated callback protocols.

<Info>
This section transforms your agent from a simple conversationalist into a fully integrated business automation tool that can take real actions and connect with your existing systems.
</Info>

## Call Transfer

Configure how your agent intelligently transfers calls to human agents, departments, or external phone systems when needed.

### Transfer Configuration

<Steps>
  <Step title="Enable Transfer Capability">
    Toggle the "Call Transfer" option to activate the transfer functionality for your agent.
  </Step>
  <Step title="Choose Transfer Destination">
    Select between traditional phone numbers or SIP trunk endpoints based on your infrastructure.
  </Step>
  <Step title="Configure Transfer Logic">
    Define when and how your agent should initiate transfers using smart prompting and multi-language templates.
  </Step>
  <Step title="Test Transfer Flow">
    Validate that transfers work correctly with your phone system before going live.
  </Step>
</Steps>

### Transfer Types

<Tabs>
  <Tab title="Phone Number">
    **Traditional Phone Transfer**
    
    Perfect for transferring to mobile phones, landlines, or traditional business numbers.

    - **Format**: Use international format (+1234567890)
    - **Best for**: Human agents, customer service departments, sales teams
    - **Reliability**: High compatibility with all phone systems
    - **Cost**: Standard telephony rates apply

    **Configuration Example:**
    ```
    Transfer Number: +1-555-123-4567
    Department: Sales Team
    ```
  </Tab>
  <Tab title="SIP Trunk">
    **SIP/VoIP Transfer**
    
    Ideal for enterprise environments with PBX systems, call centers, or VoIP infrastructure.

    - **Format**: `sip:extension@gateway_ip:port`
    - **Best for**: Call centers, PBX extensions, enterprise VoIP
    - **Benefits**: Lower costs, advanced routing, better integration
    - **Requirements**: SIP-compatible phone system

    **Configuration Examples:**
    ```
    Sales Extension: sip:100@10.0.1.50:5060
    Support Queue: sip:support@pbx.company.com:5060  
    Manager Direct: sip:200@192.168.1.100:5060
    ```
  </Tab>
</Tabs>

## Webhook Tools

Create powerful API integrations that allow your agent to interact with external systems during conversations through our sophisticated modal interface.

<Info>
**Enhanced Modal Interface**: Webhook Tools now feature an advanced configuration modal with real-time testing, parameter mapping, authentication methods, and intelligent conversation data extraction.
</Info>

### Webhook Tool Modal Interface

Configure webhook tools through our comprehensive modal interface:

<Steps>
  <Step title="Tool Identity Configuration">
    Set up the basic tool identity and activation conditions.
    
    **Required Fields:**
    - **Name**: Unique identifier (e.g., "create_lead", "check_inventory")
    - **Description**: Clear explanation of when the AI should use this tool
    - **Execution Mode**: Choose immediate or deferred execution
  </Step>
  <Step title="API Endpoint Setup">
    Configure the core API connection details.
    
    **Configuration Options:**
    - **Base URL**: Complete webhook endpoint URL
    - **HTTP Method**: GET, POST, PUT, PATCH, or DELETE
    - **Request Timeout**: 5-60 seconds (default: 20)
    - **Error Handling**: Retry logic and fallback behavior
  </Step>
  <Step title="Authentication Configuration">
    Set up secure authentication for your API endpoints.
    
    **Supported Authentication Types:**
    - Bearer Token authentication (OAuth 2.0, JWT)
    - API Key authentication (Header or Query parameter)
    - Basic Auth with username/password
    - Custom header authentication
  </Step>
  <Step title="Parameter Mapping">
    Define intelligent parameter mapping with conversation data extraction.
    
    **Parameter Configuration:**
    - **Query Parameters**: URL-encoded for GET requests
    - **Body Parameters**: JSON payload for POST/PUT/PATCH
    - **Header Parameters**: Custom HTTP headers
    - **Value Sources**: LLM extraction, static values, conversation metadata
  </Step>
</Steps>

### Advanced Parameter Configuration

Each webhook parameter supports sophisticated data mapping:

<Tabs>
  <Tab title="Parameter Properties">
    | Property | Description | Options | Example |
    |----------|-------------|---------|---------|
    | **Name** | Parameter identifier | Custom string | `customer_name` |
    | **Type** | Data validation | string, number, boolean, integer, array, object | `string` |
    | **Required** | Mandatory field | true/false | `true` |
    | **Description** | AI understanding context | Natural language | "Customer's full name from conversation" |
    | **Value Source** | Data extraction method | LLM extraction, static value, metadata | LLM prompt |
    | **Extraction Prompt** | AI instruction | Natural language | "Extract the customer's email address" |
  </Tab>
  <Tab title="Value Sources">
    **LLM Extraction**
    
    Intelligent conversation data extraction:
    ```json
    {
      "extraction_prompt": "Extract the customer's phone number from the conversation",
      "parameter_type": "string",
      "required": true
    }
    ```

    **Static Values**
    
    Fixed values for consistent data:
    ```json
    {
      "static_value": "phone_call",
      "parameter": "source",
      "type": "string"
    }
    ```

    **Conversation Metadata**
    
    Built-in conversation data:
    - Call duration and timestamps
    - Customer and agent phone numbers
    - Call direction (inbound/outbound)
    - Call status and outcome
    - Agent ID and configuration details
  </Tab>
  <Tab title="Data Types">
    **Supported Parameter Types:**

    - **string**: Text data, names, descriptions, addresses
    - **number**: Numeric values with decimal precision
    - **integer**: Whole numbers only (IDs, quantities)
    - **boolean**: true/false values for flags
    - **array**: List of values or complex objects
    - **object**: Nested data structures with properties

    **Type Validation:**
    Automatic validation ensures data consistency and provides clear error messages for type mismatches.
  </Tab>
</Tabs>

## Recall Protocol

Set up sophisticated automatic callback systems for comprehensive customer follow-up with intelligent scheduling and success detection.

<Info>
**Smart Recall System**: Intelligent callback scheduling that recognizes successful calls, respects customer preferences, and optimizes timing for maximum effectiveness while maintaining regulatory compliance.
</Info>

### Recall System Configuration

<Steps>
  <Step title="Enable Recall Protocol">
    Activate the recall system and configure maximum callback attempts (typically 3-5 attempts).
  </Step>
  <Step title="Define Success Conditions">
    Specify what constitutes a successful call that doesn't require follow-up:
    - Objective achievement (information gathered, appointment scheduled)
    - Explicit customer requests (no contact, different method preferred)
    - Call completion scenarios (full conversation, clear decline)
  </Step>
  <Step title="Configure Callback Schedule">
    Set intelligent timing intervals between recall attempts:
    - Progressive delays (15 minutes → 4 hours → next day → weekly)
    - Business hours only restrictions
    - Timezone-aware scheduling
    - Holiday and blackout date exclusions
  </Step>
  <Step title="Set Escalation Rules">
    Define how callback strategy evolves:
    - Message variation with each attempt
    - Channel switching (phone → email → SMS)
    - Human agent escalation for persistent cases
    - Automatic compliance monitoring
  </Step>
</Steps>

### Recall Timing Strategy

| Attempt | Timing | Use Case | Success Rate |
|---------|--------|----------|-------------|
| **1st Recall** | 15-30 minutes | Busy signal, no answer | 35% |
| **2nd Recall** | 2-4 hours | Same day follow-up | 25% |
| **3rd Recall** | Next business day | Professional persistence | 20% |
| **4th Recall** | 3-5 days later | Respectful follow-up | 15% |
| **5th Recall** | 1-2 weeks later | Final attempt | 10% |

### Success Detection Examples

**Objective Achievement Scenarios:**
- Customer provided required information (contact details, preferences, requirements)
- Meeting or appointment successfully scheduled with confirmation
- Purchase decision made (positive or negative with clear finality)
- Information successfully delivered (pricing, product details, service options)
- Support issue resolved or escalated to appropriate department

**Customer-Directed Scenarios:**
- Explicit request for no further contact ("Please don't call again")
- Customer scheduled their own callback time ("Call me next Tuesday")
- Successful transfer to human agent with context preservation
- Customer requested different communication method (email, text, online)

**Call Completion Scenarios:**
- Full conversation completed with all objectives addressed
- Customer showed clear disinterest with polite but firm decline
- Answering machine message successfully delivered
- Technical issues prevented meaningful conversation

### Recall Configuration Options

**Compliance and Timing:**
- **Business Hours Only**: Restrict callbacks to defined business hours
- **Time Zone Intelligence**: Respect customer's local time zone automatically
- **Day Preferences**: Avoid weekends or specific days based on business type
- **Holiday Exclusions**: Skip national holidays and company blackout dates
- **DNC Integration**: Automatic Do Not Call list checking and compliance

**Strategy Evolution:**
- **Message Variation**: Change script or approach with each attempt
- **Channel Diversification**: Transition to email or SMS after failed calls
- **Human Escalation**: Transfer persistent cases to human agents
- **Performance Analytics**: Track and optimize recall effectiveness

## Best Practices

### Integration Strategy

1. **Start Simple, Scale Complex**
   - Begin with basic call transfer and single webhook integrations
   - Gradually add sophisticated parameter mapping and multi-system connections
   - Test each integration thoroughly before adding complexity
   - Monitor performance and optimize based on real usage data

2. **Security First Approach**
   - Always use HTTPS endpoints for webhook integrations
   - Implement proper authentication and token management
   - Validate and sanitize all incoming and outgoing data
   - Regular security audits and credential rotation

3. **User Experience Optimization**
   - Design transfers to minimize customer frustration
   - Ensure context preservation across all handoffs
   - Provide clear communication about next steps
   - Monitor customer satisfaction scores post-integration

### Performance Monitoring

1. **Key Metrics Tracking**
   - Transfer success rates and context preservation
   - Webhook response times and error rates
   - Calendar booking success and no-show rates
   - Recall protocol effectiveness and compliance

2. **Continuous Improvement**
   - Regular analysis of integration performance
   - A/B testing of different configurations

   - System optimization based on usage patterns
