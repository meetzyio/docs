## Overview

The Actions & Tools section allows you to configure what your agent can do during and after calls. This includes transferring calls to human agents or SIP trunks, ending conversations intelligently, making webhook API requests, and setting up automated follow-up protocols.

## Call Transfer

Configure how your agent handles call transfers to human agents, departments, or SIP endpoints.

### Transfer Types

<Tabs>
  <Tab title="Phone Number">
    Transfer calls to traditional phone numbers or mobile devices.
  </Tab>
  <Tab title="SIP Trunk">
    Transfer calls to SIP endpoints for integration with VoIP systems.
  </Tab>
</Tabs>

### Phone Number Transfer

<Steps>
  <Step title="Enable Transfer">
    Toggle the "Call Transfer" option to activate transfer functionality.
  </Step>
  <Step title="Select Transfer Type">
    Choose "Phone Number" from the transfer type selector.
  </Step>
  <Step title="Enter Phone Number">
    Input the destination phone number in international format (e.g., +1234567890).
  </Step>
  <Step title="Configure Transfer Prompt">
    Define when the agent should initiate transfers using the prompt field.
  </Step>
</Steps>

### SIP Transfer Configuration

For advanced VoIP integrations, you can configure SIP-based transfers:

<Steps>
  <Step title="Select SIP Transfer">
    Choose "SIP Trunk" from the transfer type options.
  </Step>
  <Step title="Enter SIP URI">
    Input the SIP URI in the format: `sip:extension@gateway_ip:port`
  </Step>
  <Step title="Example Configuration">
    Example: `sip:100@78.136.102.246:5060`
  </Step>
</Steps>

<Info>
SIP transfers are ideal for integration with PBX systems, call centers, and enterprise VoIP infrastructure.
</Info>

## Webhook Tools

Create custom API integrations that your agent can trigger during conversations.

### Creating a Webhook Tool

<Steps>
  <Step title="Add New Tool">
    Click "Add Webhook Tool" to create a new integration.
  </Step>
  <Step title="Configure Basic Information">
    - Set the tool name and description
    - The description helps the AI understand when to use this tool
  </Step>
  <Step title="Set API Endpoint">
    Enter the webhook URL that will receive the HTTP request.
  </Step>
  <Step title="Choose HTTP Method">
    Select the appropriate method: GET, POST, PUT, or DELETE.
  </Step>
  <Step title="Define Parameters">
    Add query parameters and/or request body fields as needed.
  </Step>
</Steps>

### Parameter Configuration

Each webhook parameter supports:

| Property | Description | Options |
|----------|-------------|---------|
| Name | Parameter identifier | Custom string |
| Type | Data type | string, number, boolean, integer |
| Required | Whether parameter is mandatory | true/false |
| Description | What the parameter represents | Custom text |
| Value Type | How value is determined | LLM prompt, static value |

### Query Parameters vs Body Parameters

<Tabs>
  <Tab title="Query Parameters">
    Added to the URL as query string parameters:
    ```
    GET https://api.example.com/users?name=John&age=25
    ```
  </Tab>
  <Tab title="Body Parameters">
    Sent in the request body (for POST, PUT, PATCH):
    ```json
    {
      "name": "John",
      "age": 25,
      "email": "john@example.com"
    }
    ```
  </Tab>
</Tabs>

### Advanced Webhook Settings

<AccordionGroup>
  <Accordion title="Response Timeout" icon="clock">
    Configure how long to wait for the API response (default: 20 seconds).
  </Accordion>
  <Accordion title="Interruption Control" icon="pause">
    - **Enabled**: Agent pauses while waiting for webhook response
    - **Disabled**: Agent continues speaking while request processes
  </Accordion>
  <Accordion title="Pre-tool Speech" icon="volume-high">
    Force the agent to speak before executing the webhook tool.
  </Accordion>
  <Accordion title="Tool Call Sound" icon="bell">
    Choose sound effect during webhook execution:
    - **Typing**: Typing sound effect
    - **None**: No sound effect
  </Accordion>
  <Accordion title="Execution Mode" icon="play">
    - **Immediate**: Execute during the call
    - **Deferred**: Execute after the call ends
  </Accordion>
</AccordionGroup>


### Authentication

Configure webhook authentication using request headers:

```json
{
  "request_headers": {
    "Authorization": "Bearer your-api-token",
    "Content-Type": "application/json",
    "X-API-Key": "your-api-key"
  }
}
```

## Recall Protocol

Set up automatic follow-up calls when certain conditions aren't met during the initial conversation.

### Recall Configuration

<Steps>
  <Step title="Enable Recall Protocol">
    Toggle the recall feature to activate automatic callbacks.
  </Step>
  <Step title="Set Maximum Attempts">
    Define how many retry calls should be attempted (typically 2-5).
  </Step>
  <Step title="Configure Timing">
    Set intervals between recall attempts using the schedule options.
  </Step>
  <Step title="Define Success Conditions">
    Specify what constitutes a successful call that doesn't require follow-up.
  </Step>
</Steps>

### Recall Schedule Options

| Interval | Use Case |
|----------|----------|
| 5 minutes | Immediate follow-up for urgent matters |
| 15 minutes | Quick retry for busy signals |
| 30 minutes | Standard short-term follow-up |
| 1-2 hours | Business hour follow-up |
| 4 hours | Same-day callback |
| Next business day | Professional follow-up timing |

### Success Conditions

Define scenarios that indicate a successful call:

- Customer provided required information
- Meeting or appointment was scheduled  
- Purchase decision was made
- Customer explicitly requested no follow-up
- Call was transferred to human agent

<Warning>
Use recall protocols responsibly. Excessive callbacks can annoy customers and may violate telecommunications regulations like TCPA in the US or GDPR in Europe.
</Warning>

## Best Practices

### For Call Transfers
1. **Clear Trigger Conditions** - Define specific scenarios that warrant human intervention
2. **Provide Context** - Ensure transferred calls include relevant conversation context  
3. **Test Transfer Paths** - Verify both phone and SIP transfers work reliably
4. **Have Fallback Options** - Configure backup numbers for failed transfers

### For Webhook Tools
1. **Descriptive Names** - Use clear, descriptive names so the AI knows when to trigger tools
2. **Handle API Errors** - Ensure your endpoints return user-friendly error messages
3. **Secure Endpoints** - Always use authentication headers for sensitive APIs
4. **Test Thoroughly** - Validate webhook integrations using the Playground
5. **Monitor Performance** - Track webhook response times and success rates

### For End Call Strategy
1. **Multilingual Support** - Configure detection phrases in your target languages
2. **Cultural Sensitivity** - Adapt goodbye messages to local business customs
3. **Clear Termination** - Ensure agents clearly signal call ending to customers
4. **Log Call Outcomes** - Track why calls ended for optimization insights

### For Recall Protocol
1. **Respect Boundaries** - Honor customer preferences about follow-up timing
2. **Legal Compliance** - Follow local regulations about automated callbacks
3. **Diminishing Returns** - Use fewer attempts with longer intervals over time
4. **Success Tracking** - Monitor recall effectiveness and adjust strategies
