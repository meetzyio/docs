## Overview

Meetzy's collaborative editing system enables multiple team members to work on the same AI agent simultaneously without conflicts. Built with a Git-inspired workflow, it provides real-time synchronization, change tracking, and intelligent conflict resolution.

<Info>
Collaborative editing works like a "particle accelerator for teamwork" - multiple users can contribute to an agent's development while maintaining version integrity and preventing overwrites.
</Info>

## Collaboration Features

### Real-Time Collaboration Status

The editor displays live collaboration information in the header:

<Steps>
<Step title="Status Indicators">
Monitor collaboration state through visual indicators:

- **Clean**: No local or remote changes
- **Modified**: You have unsaved local changes  
- **Conflicts**: Remote changes conflict with your edits
- **Syncing**: Changes are being synchronized
- **Locked**: Other users are actively editing
</Step>

<Step title="Collaborator Presence">
See who else is working on the agent:

```javascript
// Active collaborators shown in status
collaborators: [
  { id: "user_123", name: "Sarah Chen", section: "system-prompt" },
  { id: "user_456", name: "Mike Johnson", section: "input-params" }
]
```
</Step>

<Step title="Version Information">
Track versions with semantic numbering:
- **Local Version**: Your current working version
- **Remote Version**: Latest committed version
- **Version Diff**: Changes between versions
</Step>
</Steps>

<Tabs>
<Tab title="Local Changes">

```typescript

checkForUpdates() {
```

### Committing Your Changes

Share your changes with the team through commits:

<Steps>
<Step title="Review Changes">
The Git Changes panel shows all your modifications:
- Modified fields highlighted
- Change summaries
- Diff previews
</Step>

<Step title="Commit with Message">
Provide descriptive commit messages:

```typescript
commitChanges(message: string, deployToProduction: boolean) {
  this.commitInProgress = true;
  
  const commit = {
    message: message,
    changes: Array.from(this.localChanges.entries()),
    timestamp: new Date().toISOString(),
    deploy: deployToProduction

  this.syncChanges(commit);

```

<Step title="Deployment Options">

- **Commit Only**: Save changes for review
- **Commit & Deploy**: Push changes live instantly

</Steps>

### Conflict Resolution

When multiple users edit the same fields, conflicts must be resolved:

<Warning>
Conflicts occur when two users modify the same part of an agent simultaneously. The system locks editing until conflicts are resolved.
</Warning>

<Steps>
<Step title="Conflict Detection">
The system automatically detects conflicts:

```typescript
// Conflict indicators
gitStatus = 'conflict';
conflictFields = ['system_prompt', 'input_parameters'];
```
</Step>

<Step title="Resolution Interface">
Choose how to resolve each conflict:
- **Accept Yours**: Keep your local changes
- **Accept Theirs**: Use the remote changes
- **Merge**: Combine both versions intelligently
- **Manual Edit**: Create a custom resolution
</Step>

<Step title="Complete Resolution">
Once all conflicts are resolved, commit the merged changes:

```typescript
resolveConflicts(resolutions: ConflictResolution[]) {
  resolutions.forEach(resolution => {
    this.applyResolution(resolution);
  });
  
  this.gitStatus = 'modified';
  this.conflictFields = [];
}
```
</Step>
</Steps>

## Auto-Sync Features

### Background Synchronization

The editor automatically syncs changes in the background:

<Info>
Auto-sync runs every 30 seconds to check for remote updates and push local changes.
</Info>

```typescript
// Auto-sync configuration
private setupAutoSync() {
  this.autoSyncInterval = setInterval(() => {
    if (this.gitStatus !== 'syncing' && this.gitStatus !== 'locked') {
      this.checkForUpdates();
    }
  }, 30000); // Every 30 seconds
}
```

### Intelligent Merging

The system intelligently merges non-conflicting changes:

<Tabs>
<Tab title="Safe Merges">
Changes to different fields merge automatically:

```typescript
// Safe merge example
localChanges = { system_prompt: 'updated prompt' };
remoteChanges = { voice_settings: { speed: 1.2 } };

// Result: Both changes preserved
mergedState = {
  system_prompt: 'updated prompt',
  voice_settings: { speed: 1.2 }
};
```
</Tab>

<Tab title="Field-Level Conflicts">
Conflicts only occur for the same field:

```typescript
// Conflict scenario
localChanges = { system_prompt: 'version A' };
remoteChanges = { system_prompt: 'version B' };

// Requires manual resolution
conflictFields = ['system_prompt'];
```
</Tab>
</Tabs>

## Collaboration Best Practices

### Team Workflow

<Steps>
<Step title="Start with Latest">
Always pull the latest changes before starting work:
1. Open the agent editor
2. Click "Check" for updates
3. Pull any remote changes
4. Begin your edits
</Step>

<Step title="Communicate Changes">
Use descriptive commit messages:
- ✅ "Updated greeting to be more conversational"
- ✅ "Added input parameter for customer ID"
- ❌ "misc changes"
- ❌ "update"
</Step>

<Step title="Frequent Commits">
Commit changes regularly to avoid conflicts:
- Commit logical units of work
- Don't wait until everything is "perfect"
- Small, frequent commits are easier to merge
</Step>
</Steps>

### Avoiding Conflicts

<Tips>
- **Work on different sections** when possible
- **Communicate with teammates** about what you're editing
- **Pull changes frequently** to stay up-to-date
- **Commit small changes often** rather than large batches
</Tips>

### Emergency Procedures

If you encounter issues:

<Warning>
If the system becomes unresponsive during collaboration:
1. Refresh the page to reset state
2. Pull latest changes to sync
3. Contact support if conflicts persist
</Warning>
