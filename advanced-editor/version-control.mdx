The Advanced Editor features a powerful integrated Git-based version control system that enables teams to collaborate on AI agents safely and efficiently. The system tracks changes in real-time, manages conflicts intelligently, and ensures data integrity across multiple users - functioning like a sophisticated code repository specifically designed for AI agent development.

## Version Control Interface

The version control interface is prominently displayed in the top-right section of the Advanced Editor toolbar, providing instant visibility into collaboration status and version information.

<Steps>
  <Step title="Real-Time Git Status Indicator">
    Shows the current repository state with intuitive color-coded badges:
    - **Clean** (Green): No local changes, fully synchronized with remote
    - **Modified** (Yellow): Local changes detected with precise count in parentheses
    - **Conflicts** (Red): Merge conflicts requiring manual resolution
    - **Syncing** (Blue): Currently synchronizing with remote repository
    - **Locked** (Orange): Another user is actively editing, preventing concurrent modifications
  </Step>

  <Step title="Version Information Display">
    Displays current local version number in clean monospace format (e.g., `v42`)
  </Step>

  <Step title="Three-Button Control System">
    Primary actions: **Check**, **Pull**, and **Commit** with intelligent state management
  </Step>
</Steps>

## Git Status States & Real-Time Indicators

The system maintains five distinct collaboration states with precise behavioral definitions and visual feedback.

### Clean State
```
Status: Clean
Version: v42
Indicator: Green badge with solid circle
```

**Characteristics:**
- No local modifications detected across any configuration section
- Fully synchronized with remote repository
- All version control actions available
- Safe environment for new edits or context switching
- Automatic background checks every 30 seconds maintain awareness

### Modified State
```
Status: Modified (3)
Version: v42
Indicator: Yellow badge showing exact change count
```

**Real-Time Change Tracking:**
- Detects modifications across system prompts, input parameters, output settings, evaluations, and agent configuration
- Live counter updates as you type or modify settings
- Changes persist through navigation within the editor
- Background synchronization detects conflicts before they occur

<Info>
The system tracks granular changes including character-level edits in prompts, parameter additions/modifications, webhook URL changes, evaluation criteria updates, and agent metadata modifications.
</Info>

### Conflict State
```
Status: Conflicts (2)
Version: v42
Indicator: Red badge with conflict count
```

**Conflict Resolution Features:**
- Field-by-field conflict identification and highlighting
- Side-by-side comparison interface with syntax highlighting
- Three resolution options: Keep Local, Keep Remote, or Manual Merge
- Automatic validation ensures merged results maintain agent functionality
- Commit operations disabled until all conflicts resolved

### Syncing State
```
Status: Syncing...
Version: v42
Indicator: Blue badge with animated loading
```

**Synchronization Process:**
- All version control buttons temporarily disabled
- Progress indicators show real-time sync status
- Automatic safety timeout after 30 seconds prevents hanging
- Background processes continue during sync operations

### Locked State
```
Status: Locked by 2 user(s)
Version: v42
Indicator: Orange badge with collaborator count
```

**Collaborative Lock Management:**
- Prevents simultaneous editing conflicts
- Pull operations disabled to maintain consistency
- Check operations remain available for monitoring
- Real-time updates when collaborators finish editing

<Warning>
Locked state indicates active collaboration. The system automatically releases locks when users navigate away or become inactive, typically within 60 seconds.
</Warning>

## Troubleshooting Common Issues

### Stuck in Syncing State

If status remains "Syncing..." beyond the 30-second timeout:

<Steps>
  <Step title="Automatic Recovery Wait">
    Allow system's automatic recovery timeout to complete (30 seconds)
  </Step>

  <Step title="Network Diagnostics">
    Verify internet connectivity and Meetzy API service status
  </Step>

  <Step title="Editor Refresh">
    Reload the Advanced Editor page to reset connection state
  </Step>

  <Step title="Browser Console Check">
    Open developer tools to check for JavaScript errors or network issues
  </Step>

  <Step title="Support Escalation">
    If issue persists, contact support with specific version numbers and error details
  </Step>
</Steps>

### Lost Changes During Pull Operations

If local changes disappear unexpectedly after pulling remote updates:

**Recovery Process:**
1. **Conflict Resolution Review**: Check the Changes tab for merge results and resolution status
2. **Browser Console Analysis**: Look for JavaScript errors or network interruptions during merge
3. **Version History Audit**: Verify changes weren't automatically merged in unexpected ways
4. **Backup Recovery**: Contact support with before/after version numbers for backup restoration

<Warning>
While the system maintains automatic backups during pull operations, complex merges involving significant conflicts may require manual recovery assistance from the support team.
</Warning>

### Version Number Misalignment

When local and remote versions show unexpected differences or synchronization issues:

```typescript
// Diagnostic Data Collection
const diagnosticInfo = {
  localVersion: 42,
  remoteVersion: 45,  
  gitStatus: "Modified (3)",
  lastSyncTimestamp: "2024-01-15T14:30:00Z",
  pendingChanges: ["system_prompt", "input_params", "evaluations"],
  conflictFields: [],
  collaboratorCount: 2
}
```

**Systematic Resolution Steps:**
<Steps>
  <Step title="State Documentation">
    Document current state with version numbers, status, and pending changes
  </Step>

  <Step title="Change Preservation">
    Save local changes as backup (copy critical modifications to external document)
  </Step>

  <Step title="Fresh Synchronization">
    Perform fresh Pull operation to force resynchronization with remote
  </Step>

  <Step title="Manual Reapplication">
    Manually reapply any lost modifications using backup documentation
  </Step>

  <Step title="Validation Commit">
    Commit restored changes with descriptive message explaining the recovery
  </Step>
</Steps>

### Persistent Conflict States

When conflicts remain unresolvable through normal resolution tools:

**Advanced Resolution Techniques:**
- **Field-by-Field Analysis**: Examine each conflicted field individually for resolution patterns
- **Version History Review**: Check recent commits to understand the source of conflicting changes
- **Team Consultation**: Coordinate with teammates who made conflicting changes for guided resolution
- **Fresh Start Option**: In extreme cases, create new agent version and manually migrate working components
