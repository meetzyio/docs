## Números de Teléfono en Meetzy

Cada agente de Meetzy necesita al menos un número de teléfono para:

- **Recibir llamadas entrantes**
- **Realizar llamadas salientes**
- **Identificarse** (caller ID)

<Info>
  Los números de teléfono son recursos exclusivos. Cada número solo puede estar asignado a un agente a la vez.
</Info>

## Tipos de Números de Teléfono

Meetzy soporta dos tipos de números telefónicos:

### Números Twilio (Propiedad de Meetzy)

- **Administrados por Meetzy**: Números comprados y gestionados por la plataforma
- **Uso ilimitado**: Sin restricciones de volumen de llamadas
- **Intercambio automático**: Disponibles para función de swap automático
- **Indicador visual**: Aparecen sin marcadores especiales en la interfaz

### Caller IDs Verificados (Propiedad del Cliente)

- **Números del cliente**: Números telefónicos propios verificados por Twilio
- **Mayor confianza**: Los destinatarios reconocen números conocidos
- **Control total**: El cliente mantiene la propiedad y control del número
- **Indicador visual**: Aparecen marcados como **"(verified)"** en la interfaz

<Tabs>
  <Tab title="Números Twilio">
    ```
    +34 612 345 678 (Equipo de Ventas)
    ✅ Disponible para swap
    ✅ Sin límites de uso
    ✅ Gestión automática
    ```
  </Tab>
  <Tab title="Caller IDs Verificados">
    ```
    +34 987 654 321 (verified) (Línea Principal)
    🔒 Propiedad del cliente
    📞 Número conocido por destinatarios
    ⚠️ No disponible para swap automático
    ```
  </Tab>
</Tabs>

### Diferencias Operativas

| Aspecto | Números Twilio | Caller IDs Verificados |
|---------|----------------|------------------------|
| **Propiedad** | Meetzy | Cliente |
| **Función Swap** | ✅ Disponible | ❌ No disponible |
| **Volumen** | Sin límites | Sin límites |
| **Identificación** | Número genérico | Número conocido |
| **Gestión** | Automática | Manual por cliente |
| **Desconexión** | Devuelve al pool | Elimina de la lista |

## Gestión de Subcuentas de Twilio

Meetzy soporta el uso de **subcuentas de Twilio** para organizar mejor tus números y facturación. Esto te permite:

- **Segregar números** por proyectos, equipos o campañas
- **Separar facturación** entre diferentes líneas de negocio
- **Aislar configuraciones** para mayor control y seguridad

<Steps>
  <Step title="Visualizar Subcuentas">
    En la sección de Phone Numbers, verás pestañas para cada subcuenta disponible:
    - **Cuenta Principal (Meetzy)**: Tu cuenta principal
    - **Subcuentas**: Organizadas por nombre de empresa
  </Step>
  <Step title="Cambiar entre Subcuentas">
    Haz clic en cualquier pestaña para ver los números de esa subcuenta específica.
  </Step>
  <Step title="Gestionar por Subcuenta">
    Cada subcuenta mantiene su propia lista de números, configuraciones y asignaciones.
  </Step>
</Steps>

<Warning>
  Los números de diferentes subcuentas no pueden mezclarse en un mismo agente. Cada agente debe usar números exclusivamente de una subcuenta.
</Warning>

## Asignar un Número al Crear un Agente

Al crear un nuevo agente en el Editor Avanzado:

<Steps>
  <Step title="Acceder a Settings">
    Ve a la sección **Settings** (icono de Engranaje) en el editor.
  </Step>
  <Step title="Seleccionar Número">
    En el campo **Phone Number**, selecciona un número disponible del dropdown.
    - Los números Twilio aparecen sin marcadores
    - Los Caller IDs verificados aparecen con "(verified)"
  </Step>
  <Step title="Confirmar Asignación">
    El número quedará asignado exclusivamente a este agente.
  </Step>
</Steps>

### Si No Hay Números Disponibles

Si el dropdown está vacío, verás un mensaje:

```
No available phone numbers for the selected account.
All numbers are assigned or configured.
```

**Opciones disponibles:**
- **Liberar un número** de otro agente
- **Solicitar nuevos números** mediante el enlace "Request more numbers"
- **Verificar caller IDs** propios a través del proceso de verificación
- **Cambiar de subcuenta** si tienes acceso a otras con números disponibles

## Ver Números Asignados

Una vez asignado, el número aparece en modo solo lectura:

<Tabs>

    Phone number assigned to this agent

  </Tab>

<Warning>

## Estados de Asignación

Los números pueden tener diferentes estados de asignación:

<Tabs>
  <Tab title="Assigned">
    **Completamente asignado** a un agente específico.
    - ✅ Disponible para llamadas entrantes y salientes
    - ✅ Identificación clara del agente propietario
  </Tab>
  <Tab title="Partial">
    **Parcialmente configurado** - tiene webhook pero sin agente específico.
    - ⚠️ Funcional pero requiere atención
    - 🔧 Puede necesitar asignación completa
  </Tab>
  <Tab title="Unassigned">
    **Sin asignar** - disponible para uso.
    - 💡 Listo para ser asignado a cualquier agente
    - 📞 No recibe ni realiza llamadas hasta ser asignado
  </Tab>
</Tabs>

## Múltiples Números por Agente

Un agente puede tener varios números asignados, incluyendo mezclas de números Twilio y Caller IDs verificados:

### Beneficios de Múltiples Números

- **Distribución de volumen**: Reducir riesgo de spam
- **Segmentación regional**: Diferentes números para diferentes áreas
- **Redundancia**: Respaldo si un número tiene problemas
- **Campañas específicas**: Números dedicados por campaña
- **Mayor confianza**: Combinar números conocidos con números de respaldo

### Comportamiento con Múltiples Números

| Tipo de Llamada | Comportamiento |
|-----------------|----------------|
| **Entrante** | El cliente puede llamar a cualquiera de los números |
| **Saliente** | El sistema selecciona automáticamente el mejor número disponible |

**Prioridad para llamadas salientes:**
1. Caller IDs verificados (mayor confianza)
2. Números Twilio con mejor Phone Health
3. Números disponibles por orden de asignación

## Verificación de Caller IDs

Para usar tus propios números telefónicos como Caller IDs:

<Steps>
  <Step title="Acceder a Verificación">
    En el Editor Avanzado, sección Settings, busca "Add Caller ID".
  </Step>
  <Step title="Introducir Número">
    Ingresa el número telefónico propio en formato internacional (+34612345678).
  </Step>
  <Step title="Recibir Llamada de Verificación">
    Twilio realizará una llamada automática al número proporcionado.
  </Step>
  <Step title="Introducir Código">
    Anota el código de verificación de 6 dígitos y introdúcelo en el formulario.
  </Step>
  <Step title="Completar Verificación">
    Una vez verificado, el número aparecerá disponible como "(verified)" en el dropdown.
  </Step>
</Steps>

### Estados de Caller ID

- **Pending**: Verificación en proceso
- **Verified**: Listo para usar en llamadas salientes
- **Disabled**: Temporalmente deshabilitado

<Info>
  Los Caller IDs verificados deben ser números telefónicos reales bajo tu control. No pueden ser números gratuitos ni líneas especiales.
</Info>

## Intercambiar Números (Swap)

Puedes intercambiar un número Twilio problemático por uno nuevo desde el reporte de Phone Health:

<Steps>
  <Step title="Abrir Phone Health">
    Haz clic en el indicador de salud del agente.
  </Step>
  <Step title="Identificar el número">
    Busca el número Twilio con estado Warning o Spam.
  </Step>
  <Step title="Iniciar el Swap">
    Haz clic en el botón **Swap** junto al número problemático.
  </Step>
  <Step title="Confirmar intercambio">
    El sistema muestra el nuevo número propuesto. Confirma el intercambio.
  </Step>
</Steps>

### Limitaciones del Swap

<Warning>
  La función de swap solo está disponible para números Twilio. Los Caller IDs verificados no pueden ser intercambiados automáticamente ya que son propiedad del cliente.
</Warning>

### Qué Sucede Durante el Swap

1. **Número Twilio antiguo** se desvincula y marca como "DEVOLVER"
2. **Número Twilio nuevo** se asigna y está activo inmediatamente
3. **Caller IDs verificados** permanecen inalterados
4. **Configuraciones** se transfieren al nuevo número Twilio
5. **Historial** se mantiene asociado al agente

## Gestión desde Phone Health

El reporte de Phone Health muestra información detallada sobre todos los números del agente:

### Información Mostrada

Para cada número se muestra:

- **Número telefónico** y tipo (Twilio o verified)
- **Estado de salud** (Healthy, Warning, Spam)
- **Estadísticas de llamadas** (total, completadas, spam ratio)
- **Acciones disponibles** según el tipo de número

### Acciones Disponibles por Tipo

<Tabs>
  <Tab title="Números Twilio">
    - **Swap**: Intercambiar por un número nuevo
    - **Disconnect**: Desconectar y devolver al pool
    - **View Details**: Ver estadísticas detalladas
  </Tab>
  <Tab title="Caller IDs Verificados">
    - **Disconnect**: Eliminar de la lista del agente
    - **View Details**: Ver estadísticas detalladas
    - ❌ **Swap no disponible** (son propiedad del cliente)
  </Tab>
</Tabs>

## Configuración de Transferencia de Llamadas

Los agentes pueden configurarse para transferir llamadas a números telefónicos o troncos SIP cuando sea necesario:

### Transferencia a Número Telefónico

<Steps>
  <Step title="Habilitar Transferencia">
    En la sección **Call Transfer** del editor, activa la transferencia de llamadas.
  </Step>
  <Step title="Seleccionar Tipo">
    Elige **Phone Number** como tipo de transferencia.
  </Step>
  <Step title="Configurar Número">
    Ingresa el número de teléfono de destino en formato internacional (ej: +1234567890).
  </Step>
  <Step title="Definir Mensaje">
    Configura el mensaje que determinará cuándo transferir la llamada.
  </Step>
</Steps>

### Transferencia a SIP Trunk

<Steps>
  <Step title="Seleccionar SIP">
    Elige **SIP Trunk** como tipo de transferencia.
  </Step>
  <Step title="Configurar URI">
    Ingresa la URI SIP en formato: `sip:extension@gateway_ip:port`
    Ejemplo: `sip:100@78.136.102.246:5060`
  </Step>
  <Step title="Verificar Conectividad">
    Asegúrate de que el gateway SIP sea accesible desde los servidores de Meetzy.
  </Step>
</Steps>

### Mensajes de Transferencia por Idioma

Meetzy incluye plantillas predefinidas para diferentes idiomas:

- **Español**: Transferencia en casos de rechazo, solicitud humana o desconfianza
- **Inglés**: Transfer for rejection, human request, or distrust scenarios
- **Francés**: Transfert pour refus, demande humaine ou méfiance
- **Alemán**: Übertragung bei Ablehnung, menschlicher Anfrage oder Misstrauen
- **Italiano**: Trasferimento per rifiuto, richiesta umana o sfiducia
- **Portugués**: Transferência para rejeição, solicitação humana ou desconfiança

<Info>
  Las plantillas de transferencia se adaptan automáticamente al idioma principal del agente para mantener coherencia en la conversación.
</Info>

## Gestión Avanzada de Números

### Asignar Números Manualmente

Desde la vista de Phone Numbers puedes:

<Steps>
  <Step title="Seleccionar número no asignado">
    Identifica números con estado "Unassigned" (Twilio o verified).
  </Step>
  <Step title="Hacer clic en Assign">
    Usa el botón de asignación junto al número.
  </Step>
  <Step title="Seleccionar agente">
    Elige el agente de destino del dropdown.
  </Step>
  <Step title="Confirmar asignación">
    El número queda inmediatamente disponible para el agente.
  </Step>
</Steps>

### Desconectar un Número

Para eliminar un número del agente:

<Steps>
  <Step title="Abrir Phone Health">
    Ve al reporte de Phone Health del agente.
  </Step>
  <Step title="Localizar número">
    Encuentra el número que deseas desconectar.
  </Step>
  <Step title="Hacer clic en Disconnect">
    Usa el botón **Disconnect** junto al número.
  </Step>
  <Step title="Confirmar desconexión">
    Confirma la acción en el modal que aparece.
  </Step>
</Steps>

### Comportamiento de Desconexión

<Tabs>
  <Tab title="Números Twilio">
    - Se devuelven al pool de números disponibles
    - Pueden ser reasignados a otros agentes
    - Mantienen su configuración Twilio
  </Tab>
  <Tab title="Caller IDs Verificados">
    - Se eliminan de la lista del agente
    - Mantienen su estado de verificación en Twilio
    - Pueden ser reasignados usando el mismo proceso
  </Tab>
</Tabs>

<Warning>
  Si desconectas todos los números del agente, este no podrá hacer ni recibir llamadas hasta que asignes otro número.
</Warning>

### Devolver Números al Pool

Los números Twilio marcados como "discarded" o problemáticos pueden devolverse:

1. En la vista de Phone Numbers, filtra por números descartados
2. Selecciona **Return to Pool** para números específicos  
3. El número queda disponible para reasignación

<Info>
  Esta función solo aplica a números Twilio. Los Caller IDs verificados no forman parte del pool de Meetzy.
</Info>

## Filtrado y Búsqueda

La interfaz de Phone Numbers incluye herramientas avanzadas:

### Filtros por Estado
- **All**: Todos los números (Twilio y verified)
- **Assigned**: Solo números asignados
- **Unassigned**: Solo números disponibles
- **Discarded**: Números Twilio marcados para devolución

### Búsqueda Inteligente
Busca por:
- **Número de teléfono**: Parcial o completo
- **Nombre amigable**: Descripción del número
- **Agente asignado**: Nombre del agente
- **Tipo**: "verified" para filtrar solo Caller IDs

### Ordenamiento
Ordena la lista por:
- Número de teléfono
- Nombre amigable
- Estado de asignación
- Agente asignado
- Tipo de número (Twilio/verified)

## Solicitar Nuevos Números

Para solicitar números Twilio adicionales:

<Steps>
  <Step title="Acceder al formulario">
    En el Editor Avanzado, sección Settings, haz clic en **"Request more numbers"**.
  </Step>
  <Step title="Especificar requisitos">
    Indica cuántos números necesitas y para qué subcuenta.
  </Step>
  <Step title="Enviar solicitud">
    Serás redirigido al portal de solicitud para completar el proceso.
  </Step>
  <Step title="Esperar aprovisionamiento">
    Los números nuevos suelen estar disponibles en 24-48 horas.
  </Step>
</Steps>

<Info>
  Este proceso solo aplica a números Twilio administrados por Meetzy. Para usar tus propios números, utiliza el proceso de verificación de Caller ID.
</Info>

## Mejores Prácticas

### Estrategia de Números Mixta

Para campañas optimizadas, combina ambos tipos:

<Steps>
  <Step title="Caller ID Principal">
    Asigna tu número principal verificado para mayor reconocimiento.
  </Step>
  <Step title="Números de Respaldo">
    Añade números Twilio para distribuir volumen y evitar saturación.
  </Step>
  <Step title="Monitorear Rendimiento">
    Usa Phone Health para identificar qué tipo funciona mejor.
  </Step>
  <Step title="Rotar Números Twilio">
    Intercambia números Twilio problemáticos manteniendo los Caller IDs.
  </Step>
</Steps>

### Identificación del Número

Cada número tiene un **Friendly Name** para identificarlo:

```
+34 612 345 678 (Equipo de Ventas España)
       │                    │
       │                    └── Friendly Name descriptivo
       └── Número de teléfono

+34 987 654 321 (verified) (Línea Principal)
       │            │               │
       │            │               └── Friendly Name descriptivo
       │            └── Indicador de Caller ID verificado
       └── Número de teléfono
```

**Usa nombres descriptivos como:**
- "Número Principal (verified)"
- "Respaldo Ventas España"
- "Campaña Black Friday 2024"
- "Línea Directa CEO (verified)"

### Organización por Tipo

Organiza tus números por propósito:

- **Caller IDs verificados**: Para números principales y reconocibles
- **Números Twilio**: Para distribución de volumen y campañas específicas
- **Transferencias**: Para destinos de escalación y soporte humano

## Finalización Automática de Llamadas

Los agentes pueden configurarse para finalizar llamadas automáticamente en situaciones específicas:

### Detección de Contestador Automático

El agente puede identificar y colgar automáticamente cuando detecta:

- "En este momento no podemos atenderle"
- "Deje su mensaje después de la señal"
- "El número al que llama..."
- "Este es el contestador de..."
- "Gracias por llamar, en estos momentos..."

### Finalización Natural

El agente también finaliza llamadas cuando:
- El usuario se despide claramente
- El agente completa su despedida
- Se alcanza el objetivo de la conversación

<Info>
  La detección de contestadores automáticos utiliza análisis de patrones de voz y frases comunes para identificar con precisión estos sistemas.
</Info>

## Troubleshooting

### Números No Aparecen

Si no ves números esperados:

1. **Verifica la subcuenta** - Asegúrate de estar en la pestaña correcta
2. **Actualiza la vista** - Recarga la página o cambia de pestaña
3. **Revisa permisos** - Confirma acceso a la subcuenta específica
4. **Verifica Caller IDs** - Los números verificados aparecen con "(verified)"
5. **Contacta soporte** si el problema persiste

### Problemas de Verificación de Caller ID

Si falla la verificación:

1. **Número incorrecto**: Verifica que el número esté en formato internacional correcto
2. **Número ocupado**: Asegúrate de estar disponible para recibir la llamada de verificación
3. **Código incorrecto**: Anota cuidadosamente el código de 6 dígitos
4. **Número no válido**: Algunos tipos de números no son elegibles para verificación

### Problemas de Swap

Para números Twilio:

1. **Sin números disponibles**: Solicita más números antes de hacer swap
2. **Caller ID no elegible**: Los Caller IDs verificados no pueden ser intercambiados
3. **Permisos insuficientes**: Confirma permisos de administración

### Errores de Asignación

Si falla la asignación:

- **Número ya asignado**: Otro agente puede estar usando el número
- **Subcuenta incorrecta**: Verifica que el agente y número estén en la misma subcuenta
- **Caller ID no verificado**: Completa el proceso de verificación primero
- **Permisos insuficientes**: Confirma permisos de administración

## Preguntas Frecuentes

<AccordionGroup>
  <Accordion title="¿Puedo usar mi propio número de teléfono?">
    Sí, a través del sistema de Caller IDs verificados. Puedes verificar tus números propios y usarlos como identificación para llamadas salientes.
  </Accordion>
  
  <Accordion title="¿Cuál es la diferencia entre números Twilio y Caller IDs verificados?">
    Los números Twilio son administrados por Meetzy y pueden ser intercambiados automáticamente. Los Caller IDs verificados son tus números propios, ofrecen mayor confianza pero no pueden ser intercambiados automáticamente.
  </Accordion>
  
  <Accordion title="¿Puedo mezclar números Twilio y Caller IDs en un mismo agente?">
    Sí, puedes asignar ambos tipos al mismo agente. El sistema priorizará los Caller IDs verificados para llamadas salientes por su mayor confianza.
  </Accordion>
  
  <Accordion title="¿Por qué no puedo hacer swap de un Caller ID verificado?">
    Los Caller IDs verificados son números de tu propiedad. Meetzy no puede intercambiarlos automáticamente porque no los administra. Debes gestionarlos manualmente.
  </Accordion>
  
  <Accordion title="¿Qué pasa si se agotan mis números Twilio?">
    Las llamadas salientes priorizarán los Caller IDs verificados disponibles. Si no hay números disponibles, las llamadas fallarán hasta que solicites más números o liberes algunos existentes.
  </Accordion>
  
  <Accordion title="¿Los Caller IDs verificados tienen límites de uso?">
    No, una vez verificados pueden usarse ilimitadamente para llamadas salientes, igual que los números Twilio.
  </Accordion>
  
  <Accordion title="¿Puedo verificar números internacionales?">
    Sí, puedes verificar números de cualquier país soportado por Twilio. El proceso de verificación funciona globalmente.
  </Accordion>
  
  <Accordion title="¿Cómo mantengo la salud de mis números?">
    Usa el reporte Phone Health para monitorear el rendimiento. Para números Twilio, usa la función swap cuando sea necesario. Para Caller IDs, considera rotar su uso o verificar números adicionales.
  </Accordion>
</AccordionGroup>
