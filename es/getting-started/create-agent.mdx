## El Nuevo Flujo Prompt-First

Meetzy ahora ofrece una experiencia mejorada para crear agentes. Simplemente describe qué debe hacer tu agente y nuestra IA construirá la configuración inicial por ti.

<Info>
  **¿Nuevo en Meetzy?** El flujo prompt-first es la manera más rápida de empezar. Para usuarios avanzados, también puedes crear un agente desde cero.
</Info>

## Creación Automática desde Onboarding

Meetzy incluye un proceso completamente automatizado para nuevos usuarios que llegan desde nuestro landing page de autoservicio. Este flujo preserva todos los datos introducidos y redirige automáticamente al usuario al editor de agentes.

### Cómo Funciona la Preservación de Datos

<Steps>
  <Step title="Captura de Datos en Landing Page">
    Los usuarios completan un formulario en la página de autoservicio que captura:
    - Prompt/descripción del agente deseado
    - Canales de comunicación preferidos
    - Dirección de negocio específica
  </Step>
  
  <Step title="Almacenamiento Temporal">
    Los datos se guardan en `localStorage` con una expiración automática:
    ```javascript
    localStorage.setItem('meetzy_onboarding_prompt', JSON.stringify({
      prompt: userPrompt,
      channels: selectedChannels,
      directions: businessDirections,
      expiresAt: Date.now() + (24 * 60 * 60 * 1000) // 24 horas
    }));
    ```
  </Step>
  
  <Step title="Redirección Inteligente">
    Al completar la autenticación, el sistema detecta automáticamente si el usuario viene del onboarding:
    - Verifica datos válidos en `localStorage`
    - Redirige directamente al editor de agentes
    - Preserva el contexto completo del proceso
  </Step>
  
  <Step title="Aplicación Automática">
    El editor de agentes recupera y aplica automáticamente todos los datos preservados, generando la configuración inicial sin intervención del usuario.
  </Step>
</Steps>

### Detección de Modo Onboarding

El sistema detecta automáticamente cuando un usuario viene del proceso de onboarding:

<Tabs>
  <Tab title="URL Parameters">
    ```
    /login?source=onboarding
    ```
    Activa automáticamente el modo onboarding en la pantalla de autenticación.
  </Tab>
  
  <Tab title="Data Preservation">
    ```typescript
    // AuthService verifica datos válidos
    getOnboardingData(): { prompt: string; channels: string; directions: string } | null {
      const savedData = localStorage.getItem('meetzy_onboarding_prompt');
      if (!savedData) return null;
      
      const parsed = JSON.parse(savedData);
      if (parsed.expiresAt && parsed.expiresAt < Date.now()) {
        localStorage.removeItem('meetzy_onboarding_prompt');
        return null;
      }
      
      return {
        prompt: parsed.prompt || '',
        channels: parsed.channels || '',
        directions: parsed.directions || '',
      };
    }
    ```
  </Tab>
  
  <Tab title="Automatic Redirect">
    ```typescript
    // LoginComponent redirige automáticamente
    private navigateToDashboard(): void {
      const onboardingData = this.authService.getOnboardingData();
      if (onboardingData?.prompt) {
        this.router.navigate(['/agents/new'], {
          queryParams: { onboarding: 'true' }
        });
      } else {
        this.router.navigate(['/playbooks']);
      }
    }
    ```
  </Tab>
</Tabs>

<Info>
  **Expiración Automática**: Los datos de onboarding expiran automáticamente después de 24 horas para mantener la seguridad y evitar datos obsoletos.
</Info>

## Requisitos Previos

Antes de crear un agente, asegúrate de tener:

- Una cuenta activa de Meetzy con acceso al dashboard
- Al menos un número de teléfono disponible en tu cuenta
- Una idea clara de tu caso de uso (ventas, soporte, etc.)

## Crear un Agente con Prompt-First

<Steps>
  <Step title="Accede al Editor Avanzado">
    Desde el dashboard, haz clic en "Create Agent" o navega directamente al Editor Avanzado.
  </Step>
  
  <Step title="Describe tu Agente">
    En la pantalla inicial con fondo animado, describe qué debe hacer tu agente:
    
    ```
    Un agente que responde llamadas de atención al cliente para una clínica dental, 
    agenda citas y transfiere casos urgentes a la recepcionista...
    ```
    
    <Info>
      **Consejo**: Sé específico sobre el rol, la industria y las acciones principales que debe realizar.
    </Info>
  </Step>
  
  <Step title="Genera la Configuración">
    Presiona **Enter** o haz clic en el botón de envío. Meetzy generará automáticamente:
    - Prompt de sistema optimizado
    - Configuración básica recomendada
    - Saludos personalizados
    - Parámetros de entrada sugeridos
  </Step>
  
  <Step title="Refina y Personaliza">
    Revisa la configuración generada y ajusta según tus necesidades específicas usando las herramientas del Editor Avanzado.
  </Step>
</Steps>

<Warning>
  Si prefieres empezar desde cero, puedes hacer clic en "or start from scratch" en la pantalla inicial.
</Warning>

## Prompt de Sistema

El editor central contiene el **Prompt de Sistema** - las instrucciones principales que definen el comportamiento de tu agente.

### Estructura Recomendada

```markdown
Eres un agente amigable de cualificación de ventas para Acme Corp.

## Objetivo
Cualificar leads entendiendo sus necesidades y presupuesto,
luego agendar una demo con un representante de ventas.

## Tu Personalidad
- Profesional pero cercano
- Empático y buen oyente  
- Orientado a soluciones
- Respetuoso con el tiempo del cliente

## Proceso de Cualificación
1. Saluda calurosamente y presenta la empresa
2. Pregunta sobre sus desafíos actuales
3. Entiende su cronograma y presupuesto
4. Evalúa el ajuste con nuestras soluciones
5. Si está cualificado, agenda una demo

## Directrices de Conversación
- Sé conversacional y amigable
- Haz preguntas abiertas para entender mejor
- Escucha activamente y reformula para confirmar
- Mantén el foco en sus necesidades, no en vender

## Restricciones
- Nunca discutas precios directamente
- No hagas promesas sobre características específicas
- Transfiere a un humano si el cliente está molesto
- No interrumpas cuando el cliente está hablando

## Palabras Clave para Cualificación
- Presupuesto: "inversión", "costo", "precio"
- Urgencia: "cuándo", "pronto", "urgente" 
- Autoridad: "decisión", "aprobación", "jefe"
- Necesidad: "problema", "desafío", "mejorar"
```

### Consejos para un Prompt Efectivo

<Tips>
  <Tip title="Sé Específico">
    Define claramente el rol, objetivos y personalidad del agente
  </Tip>
  
  <Tip title="Usa Ejemplos">
    Incluye ejemplos de respuestas apropiadas e inapropiadas
  </Tip>
  
  <Tip title="Define Límites">
    Especifica qué debe y no debe hacer el agente
  </Tip>
  
  <Tip title="Incluye Contexto">
    Proporciona información sobre la empresa y los productos/servicios
  </Tip>
</Tips>

**Características:**

- Prueba diferentes escenarios

Ejecuta pruebas automatizadas para validar el comportamiento:

**Tipos de pruebas:**
- **Funcionales**: Verifica que las acciones funcionen
- **Conversacionales**: Evalúa la calidad de las respuestas
- **De Rendimiento**: Mide tiempos de respuesta
- **De Seguridad**: Confirma que los guardrails funcionan

### Changes (📝)

Sistema de control de versiones similar a Git:

- **Ver Cambios**: Revisa modificaciones pendientes
- **Comparar Versiones**: Ve diferencias entre versiones
- **Historial**: Accede a versiones anteriores

## Publicar Tu Agente

<Steps>
  <Step title="Validar Configuración">
    Haz clic en **Check** para validar tu configuración. El sistema verificará:
    - Prompt de sistema completo
    - Número de teléfono asignado
    - Configuración de voz válida
    - Parámetros obligatorios
  </Step>
  
  <Step title="Probar en Playground">
    Usa el Playground para simular conversaciones:
    - Prueba diferentes tipos de clientes
    - Verifica el flujo de conversación
    - Confirma que las acciones funcionan
  </Step>
  
  <Step title="Ejecutar Tests">
    Ejecuta las pruebas automatizadas:
    - Tests funcionales
    - Tests de calidad conversacional
    - Validación de seguridad
  </Step>
  
  <Step title="Llamadas de Prueba">
    Realiza llamadas telefónicas reales de prueba:
    - Usa "Test Agent" desde el dashboard
    - Llama al número asignado
    - Valida la experiencia completa
  </Step>
  
  <Step title="Publicar a Producción">
    Haz clic en **Publish** para desplegar:
    - Se crea una nueva versión
    - El agente queda disponible para llamadas
    - La versión anterior se mantiene para rollback
  </Step>
</Steps>

<Warning>
  **Importante**: Publicar crea una nueva versión. La versión anterior permanece disponible para rollback si es necesario.
</Warning>

## Solución de Problemas

### Errores Comunes

<Tabs>
  <Tab title="Validación Fallida">
    **Problema**: El agente no pasa la validación
    
    **Soluciones**:
    - Verifica que el prompt no esté vacío
    - Confirma que hay un número de teléfono asignado
    - Revisa que la configuración de voz sea válida
  </Tab>
  
  <Tab title="Respuestas Inconsistentes">
    **Problema**: El agente responde de manera impredecible
    
    **Soluciones**:
    - Haz el prompt más específico y claro
    - Añade más ejemplos de respuestas esperadas
    - Reduce la complejidad del rol del agente
  </Tab>
  
  <Tab title="Problemas de Audio">
    **Problema**: La voz suena robótica o poco natural
    
    **Soluciones**:
    - Ajusta la velocidad de habla
    - Reduce la estabilidad para más variación
    - Prueba una voz diferente
  </Tab>
</Tabs>

### Optimización de Rendimiento

<Tabs>
  <Tab title="Latencia">
    - Usa modelos más rápidos (Gemini Flash, GPT-4o Mini)
    - Simplifica el prompt del sistema
    - Reduce el tamaño de la knowledge base
  </Tab>
  
  <Tab title="Precisión">
    - Usa modelos más capaces (GPT-4, Gemini Pro)
    - Añade más contexto y ejemplos
    - Implementa evaluaciones automáticas
  </Tab>
</Tabs>
