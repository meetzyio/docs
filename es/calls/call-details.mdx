# Detalles de Llamada

Cada llamada genera datos detallados que te ayudan a entender qué sucedió durante la conversación. El modal de Detalles de Llamada proporciona acceso a toda la información capturada.

## Abrir Detalles de Llamada

Haz clic en cualquier fila de la tabla de Historial de Llamadas, o haz clic en el botón **View** para abrir el modal de detalles.

## Diseño del Modal

El modal de Detalles está organizado en un layout de tres columnas:

| Panel | Contenido |
|-------|-----------|
| ⬅️ Izquierda | Parámetros de Entrada (JSON) |
| 📊 Centro | Información de la Llamada |
| ➡️ Derecha | Respuesta de Salida (JSON) |

## Parámetros de Entrada

El panel izquierdo muestra el JSON de entrada enviado para iniciar la llamada:

```json
{
  "phone": "+1234567890",
  "customer_name": "Juan García",
  "appointment_date": "2024-02-15",
  "custom_fields": {
    "order_id": "ORD-12345"
  }
}
```

### Funcionalidades

- **Botón Copiar** - Copia todo el JSON al portapapeles
- **Resaltado de Sintaxis** - JSON con colores para legibilidad
- **Expandible** - Scroll para payloads grandes

## Información de la Llamada

El panel central muestra metadatos clave:

| Campo | Descripción |
|-------|-------------|
| Call ID | Identificador único (monoespaciado, copiable) |
| Status | Estado actual con badge de color |
| Success | Indicador ✓ Sí o ✗ No |
| Playbook | Nombre del agente que manejó la llamada |
| Duration | Duración en formato legible |
| From | Número de teléfono origen |
| To | Número de teléfono destino |
| Email | Email del contacto (si se proporcionó) |
| Created Time | Timestamp de inicio de la llamada |
| Tags | Etiquetas aplicadas a esta llamada |

### Badges de Estado

| Estado | Color | Descripción |
|--------|-------|-------------|
| completed | 🟢 Verde | Llamada terminada normalmente |
| in-progress | 🔵 Azul | Llamada actualmente activa |
| failed | 🔴 Rojo | Llamada con error |
| transferred | 🟣 Morado | Llamada transferida |
| no-answer | ⚪ Gris | El destinatario no contestó |

## Respuesta de Salida

El panel derecho muestra el JSON de salida con todos los datos capturados:

```json
{
  "transcription": "Agente: Hola, soy Sara de...",
  "summary": "El cliente confirmó la cita para...",
  "sentiment": "positive",
  "output_params": {
    "appointment_confirmed": true,
    "preferred_time": "morning",
    "callback_requested": false
  },
  "evaluations": {
    "successful_greeting": true,
    "collected_requirements": true
  }
}
```

## Navegación al Copilot

<Info>
Desde los detalles de cualquier llamada, puedes navegar directamente al Copilot de IA con el contexto de la conversación precargado.
</Info>

### Acceso Contextual

En el panel central de información de la llamada, encontrarás un botón **"Analyze with AI"** que te permite:

<Steps>
<Step title="Navegación Contextual">
Haz clic en **"Analyze with AI"** para abrir el Copilot con el contexto de la llamada
</Step>
<Step title="Contexto Precargado">
El Copilot se abrirá automáticamente con referencia `@call:<id>` que incluye toda la transcripción
</Step>
<Step title="Análisis Inteligente">
Realiza consultas como "Analiza el sentimiento de esta llamada" o "Genera un resumen ejecutivo"
</Step>
</Steps>

### Flujo de Trabajo Integrado

El sistema proporciona una experiencia fluida entre el análisis de llamadas y la optimización de agentes:

1. **Revisar Llamada** - Examina transcripción y métricas en el modal de detalles
2. **Analizar con IA** - Un clic te lleva al Copilot con contexto completo
3. **Optimizar Agente** - Aplica mejoras directamente al agente desde el Copilot
4. **Validar Cambios** - Regresa al historial para verificar el impacto

## Información de Webhook

Si tu agente tiene un webhook configurado, la parte inferior del panel derecho muestra:

| Campo | Valor |
|-------|-------|
| Method | POST, GET, etc. |
| URL | Endpoint del webhook |
| Auth | Header de autorización (oculto) |

<Info>
La sección de webhook solo aparece si se configuró y ejecutó un webhook para esta llamada.
</Info>

## Reproducción de Grabación

Si la grabación de llamadas está habilitada:

1. Aparece un reproductor de audio en la parte superior del modal
2. Controles incluyen play/pause, seek y volumen
3. Opción de descarga disponible para acceso offline

<Warning>
Las grabaciones de llamadas se almacenan por tiempo limitado según tu plan. Descarga grabaciones importantes para almacenamiento permanente.
</Warning>

## Copiar Datos

### Botones de Copiar

- **Copy Input** - Copia todo el JSON de entrada
- **Copy Output** - Copia todo el JSON de salida
- **Copy Call ID** - Copia solo el identificador de llamada

### Feedback

Cuando copias datos:
- El botón cambia a "Copied!" con un checkmark
- Vuelve a la normalidad después de 2 segundos
