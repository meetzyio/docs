# Importar Contactos

Meetzy te permite importar contactos de forma masiva desde archivos Excel (.xlsx, .xls, .xlsm) o directamente desde Google Sheets. Esto te permite poblar rápidamente tu base de datos de contactos para campañas con características avanzadas de validación y configuración.

## Métodos de Importación

| Método | Descripción | Requisitos |
|--------|-------------|------------|
| 📊 Excel | Sube archivos .xlsx, .xls o .xlsm | Archivo hasta 10MB |
| 📗 Google Sheets | Conecta directamente a tus hojas de cálculo | Integración con cuenta Google |

## Importación desde Excel

### Subiendo un Archivo

1. Navega a **CRM** → **Import Contacts**
2. Selecciona **Import from Excel**
3. Arrastra y suelta tu archivo o haz clic para buscar
4. Formatos soportados: `.xlsx`, `.xls`, `.xlsm`
5. Tamaño máximo de archivo: 10MB

<Tip>
El nuevo sistema de arrastrar y soltar incluye indicadores visuales mejorados. El borde se vuelve verde cuando se detecta un archivo válido y muestra una vista previa instantánea de la estructura.
</Tip>

### Validación Avanzada del Archivo

Una vez subido, el sistema realiza análisis profundo:

| Análisis | Descripción |
|----------|-------------|
| Total Rows | Número de filas de datos detectadas |
| Valid Phones | Filas con números de teléfono válidos (con formato internacional) |
| Duplicates | Números duplicados con opción de deduplicación automática |
| No Phone | Filas sin número válido con sugerencias de corrección |
| Data Quality Score | Puntuación de calidad basada en completitud de campos |

<Warning>
El sistema ahora detecta automáticamente formatos de teléfono incorrectos y sugiere correcciones. Las filas con problemas graves serán marcadas para revisión manual.
</Warning>

## Proceso de Importación Optimizado

### Importación en Paralelo

El sistema ahora procesa importaciones en múltiples hilos:

1. **Validación paralela**: Verifica datos mientras se cargan más filas
2. **Procesamiento asíncrono**: No bloquea la interfaz durante importaciones grandes
3. **Reintentos automáticos**: Reintenta filas con errores temporales
4. **Checkpoint system**: Guarda progreso para recuperación en caso de interrupción

### Duración Optimizada

|--------|----------------|--------|

## Solución de Problemas Avanzada

| Problema | Causa Común | Solución Mejorada |
|----------|-------------|-------------------|
| "Billy consumption exceeded" | Límite de facturación alcanzado | Contactar soporte o ajustar límites en configuración |
| "Timezone validation failed" | Formato de zona horaria incorrecto | Usar formato estándar (ej: Europe/Madrid, America/Mexico_City) |
| "Recall protocol conflict" | Horarios de rellamada solapados | El sistema ahora auto-resuelve conflictos con sugerencias |
| "Segment capacity full" | Segmento ha alcanzado límite máximo | Crear nuevos segmentos o expandir capacidad |
| "Import queue full" | Demasiadas importaciones simultáneas | Sistema de cola inteligente - se procesará automáticamente |
| "Data quality score low" | Muchos campos vacíos o inválidos | Usar las sugerencias del analizador de calidad |

## Integrations y APIs

### Webhook de Importación

Configura webhooks para recibir notificaciones:

```json
{
  "event": "import.completed",
  "import_id": "imp_123456",
  "status": "success",
  "stats": {
    "imported": 1250,
    "updated": 45,
    "skipped": 12,
    "failed": 3
  },
  "timestamp": "2024-03-15T10:30:00Z"
}
```

### API de Estado

Consulta el progreso programáticamente:

```bash
GET /api/v1/imports/{import_id}/status
Authorization: Bearer {token}
```
