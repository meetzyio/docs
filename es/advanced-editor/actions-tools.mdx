## Resumen

La sección de Acciones y Herramientas te permite configurar qué puede hacer tu agente durante y después de las llamadas. Esto incluye transferir llamadas por teléfono o SIP, finalizar conversaciones automáticamente, crear webhooks personalizados y configurar protocolos de seguimiento automatizado.

<Info>
Esta sección incluye funcionalidades avanzadas que potencian significativamente las capacidades de tu agente conversacional.
</Info>

## Transferencia de Llamadas

Configura cómo tu agente maneja las transferencias de llamada a agentes humanos, extensiones internas o sistemas telefónicos externos.

### Tipos de Transferencia

Meetzy soporta dos tipos principales de transferencia:

<Tabs>
  <Tab title="Número de Teléfono">
    Transferir a números de teléfono tradicionales (móvil, fijo, internacional).
    
    **Formato**: `+34600123456`
    
    **Casos de uso**:
    - Transferir a agentes de soporte
    - Conectar con departamentos específicos
    - Escalar a supervisores
  </Tab>
  <Tab title="SIP Trunk">
    Transferir a través de protocolo SIP para sistemas telefónicos empresariales.
    
    **Formato**: `sip:extensión@gateway_ip:puerto`
    
    **Ejemplo**: `sip:100@78.136.102.246:5060`
    
    **Casos de uso**:
    - Transferir a extensiones internas
    - Conectar con sistemas PBX
    - Integrar con infraestructura VoIP existente
  </Tab>
</Tabs>

### Configuración de Transferencia

<Steps>
  <Step title="Activar Transferencia">
    Habilita la opción de transferencia de llamadas usando el toggle en la interfaz.
  </Step>
  <Step title="Seleccionar Tipo">
    Elige entre "Número de Teléfono" o "SIP Trunk" según tu infraestructura.
  </Step>
  <Step title="Configurar Destino">
    Introduce el número de teléfono o URI SIP de destino.
  </Step>
  <Step title="Definir Prompt">
    Establece las condiciones que deben activar la transferencia.
  </Step>
</Steps>

### Otras Condiciones de Finalización

Además de contestadores automáticos, el agente finalizará llamadas cuando:

- **Despedida Natural**: El usuario se despide de forma clara y el agente completa su despedida
- **Objetivos Cumplidos**: Se han alcanzado los objetivos específicos de la conversación
- **Timeout Configurado**: Se agota el tiempo máximo establecido para la llamada
- **Detección de Silencio**: Periodos prolongados sin respuesta del usuario
- **Transferencia Completada**: Después de transferir exitosamente la llamada

## Herramientas Webhook Personalizadas

Crea integraciones poderosas con sistemas externos mediante webhooks que tu agente puede activar durante las conversaciones en tiempo real.

### Creando una Herramienta Webhook

<Steps>
  <Step title="Añadir Nueva Herramienta">
    Haz clic en "Añadir Herramienta Webhook" para crear una nueva integración personalizada.
  </Step>
  <Step title="Configurar Información Básica">
    - **Nombre**: Identificador único de la herramienta (ej. "consultar_inventario")
    - **Descripción**: Explica detalladamente cuándo y por qué la IA debe usar esta herramienta
  </Step>
  <Step title="Establecer Endpoint">
    - **URL**: Endpoint que recibirá la petición HTTP
    - **Método**: Selecciona GET, POST, PUT, PATCH o DELETE
  </Step>
  <Step title="Configurar Parámetros">
    Define parámetros de query string y/o campos del cuerpo de la petición con tipos de dato específicos.
  </Step>
  <Step title="Ajustar Configuración Avanzada">
    Configura timeouts, interrupciones, modo de ejecución y comportamiento de audio.
  </Step>
</Steps>

### Configuración de Parámetros

Cada parámetro webhook puede configurarse con precisión:

| Propiedad | Descripción | Opciones |
|----------|-------------|----------|
| **Identificador** | Nombre del parámetro en la API | string único (ej. "customer_email") |
| **Tipo de Dato** | Formato del valor esperado | string, number, boolean, integer, array, object |
| **Requerido** | Si es obligatorio para la ejecución | true/false |
| **Descripción** | Contexto detallado para la IA | texto descriptivo que guía el uso |
| **Tipo de Valor** | Origen del dato | llm_prompt (IA genera), static (valor fijo) |
| **Valores Enum** | Opciones predefinidas | lista de valores permitidos |

### Configuración Avanzada de Webhooks

#### Control de Interrupciones

<Info>
Controla si el agente puede ser interrumpido mientras espera la respuesta del webhook. Útil para operaciones críticas que no deben ser interrumpidas.
</Info>

- **Deshabilitar Interrupciones**: El agente no puede ser interrumpido durante la ejecución
- **Permitir Interrupciones**: El usuario puede interrumpir mientras se procesa la petición (recomendado para consultas rápidas)

#### Modos de Ejecución

| Modo | Descripción | Caso de Uso | Ventajas |
|------|-------------|-------------|----------|
| **Inmediato** | Ejecuta durante la conversación | Consultas de datos, validaciones | Respuesta en tiempo real |
| **Diferido** | Ejecuta después de la llamada | Logs, notificaciones, CRM | No afecta fluidez de llamada |

#### Configuración de Audio

- **Sonido de Herramienta**: Reproduce sonido de "tecleado" durante la ejecución para dar feedback al usuario
- **Comportamiento de Audio**: Automático (reproduce siempre) o manual (controlado por la IA)
- **Forzar Síntesis Previa**: Asegura que el agente termine de hablar antes de ejecutar la herramienta

### Mejores Prácticas para Webhooks

<Steps>
  <Step title="Descripciones Claras">
    Escribe descripciones detalladas que expliquen exactamente cuándo la IA debe usar cada herramienta. Incluye contexto y ejemplos.

    - Valida y sanitiza todos los inputs
    - No expongas información sensible en logs
  </Step>
</Steps>

## Protocolo de Rellamada (Recall)

Configura llamadas de seguimiento automáticas cuando no se cumplen objetivos específicos o se requiere continuidad en la comunicación.

### Configuración de Recall

<Steps>
  <Step title="Habilitar Protocolo">
    Activa el sistema de rellamadas automáticas desde la interfaz de configuración.
  </Step>
  <Step title="Definir Intentos Máximos">
    Establece el número máximo de intentos de rellamada (recomendado: 2-3 para evitar molestias).
  </Step>
  <Step title="Configurar Intervalos">
    Define intervalos progresivos entre intentos para optimizar las probabilidades de éxito.
  </Step>
  <Step title="Establecer Condiciones">
    Especifica qué situaciones deben activar una rellamada automática.
  </Step>
</Steps>

### Estrategias de Intervalos

| Intervalo | Contexto | Ejemplo de Uso | Tasa de Éxito |
|-----------|----------|----------------|---------------|
| **5-15 minutos** | Línea ocupada, urgente | Emergencias, confirmaciones urgentes | 60-70% |
| **30 minutos - 1 hora** | No contestó, mismo día | Citas médicas, servicios urgentes | 45-55% |
| **2-4 horas** | Seguimiento comercial | Ventas, ofertas limitadas | 35-45% |
| **Siguiente día hábil** | Seguimiento profesional | Propuestas, información general | 25-35% |
| **48-72 horas** | Seguimiento a largo plazo | Nurturing, relaciones | 15-25% |

### Condiciones de Activación de Recall

Las rellamadas se pueden activar automáticamente en estos escenarios:

<Tabs>
  <Tab title="Condiciones Técnicas">
    - **No Response**: No se obtuvo respuesta (timbre sin contestar)
    - **Busy Signal**: Línea ocupada o señal de ocupado
    - **Voicemail**: Se detectó contestador automático
    - **Network Issues**: Problemas de conectividad o calidad de llamada
    - **Call Dropped**: Llamada cortada inesperadamente
  </Tab>
  <Tab title="Condiciones de Negocio">
    - **Incomplete Objective**: No se completó el objetivo principal de la llamada
    - **Customer Request**: Cliente solicitó ser contactado más tarde
    - **Information Pending**: Información pendiente de confirmación o recopilación
    - **Follow-up Required**: Se requiere seguimiento por el tipo de interacción
    - **Partial Completion**: Conversación incompleta por interrupciones
  </Tab>
  <Tab title="Condiciones Personalizadas">
    - **Business Hours**: Solo rellamar en horarios comerciales específicos
    - **Customer Timezone**: Respetar zona horaria del cliente
    - **Campaign Rules**: Reglas específicas de la campaña o playbook
    - **Lead Score**: Basado en puntuación o calificación del contacto
    - **Previous Interactions**: Historial de interacciones previas
  </Tab>
</Tabs>

<Warning>
**Cumplimiento Legal**: Usa protocolos de rellamada responsablemente. Demasiados callbacks pueden molestar a los clientes y violar regulaciones de telecomunicaciones. Siempre respeta las listas de "No Llamar", las preferencias del cliente, y regulaciones locales como GDPR, LOPD, o CAN-SPAM.
</Warning>

    - Verifica horarios de atención de agentes humanos
    - Prueba conectividad SIP si usas transferencias internas
  </Step>
  <Step title="Contexto Completo">
    - Asegura que el agente receptor tenga toda la información de la llamada
    - Incluye resumen de la conversación hasta el momento de transferencia
    - Proporciona razón específica de la transferencia
  </Step>
  <Step title="Escalación Estructurada">

  </Step>

  </Step>
  <Step title="Seguridad Robusta">

    - Registra logs de auditoría

## Solución de Problemas Avanzada

<Tabs>
  <Tab title="Transferencias">
    **Problema**: La transferencia falla constantemente
    
    **Diagnóstico**:
    ```bash
    # Verificar conectividad SIP
    sip-trace your-sip-gateway.com 5060
    
    # Testear número de teléfono
    curl -X POST "https://api.meetzy.com/test-transfer" \
      -H "Content-Type: application/json" \
      -d '{"number": "+34600123456"}'
    ```
    
    **Soluciones**:
    - Verificar formato correcto del número (E.164)
    - Confirmar permisos de transferencia en el sistema
    - Testear conectividad con el destino
    - Revisar configuración de firewall para SIP
    - Validar credenciales de autenticación
  </Tab>
  <Tab title="Webhooks">
    **Problema**: Webhooks lentos o con fallos
    
    **Diagnóstico**:
    ```bash
    # Testear latencia del endpoint
    curl -w "@curl-format.txt" -o /dev/null -s \
      "https://your-api.com/endpoint"
    
    # Verificar SSL
    openssl s_client -connect your-api.com:443
    ```
    
    **Soluciones**:
    - Optimizar queries de base de datos
    - Implementar caching para consultas frecuentes
    - Configurar CDN para recursos estáticos
    - Aumentar timeout gradualmente
    - Implementar circuit breaker pattern
  </Tab>
  <Tab title="Recalls">
    **Problema**: Recalls no se ejecutan o fallan
    
    **Diagnóstico**:
    ```json
    {
      "recall_status": "failed",
      "error_code": "INVALID_PHONE_FORMAT",
      "last_attempt": "2025-01-20T10:30:00Z",
      "attempts_remaining": 2
    }
    ```
    
    **Soluciones**:
    - Validar formato de números telefónicos
    - Verificar configuración de intervalos
    - Comprobar límites de intentos
    - Revisar horarios permitidos
    - Validar condiciones de activación
  </Tab>
</Tabs>
