## Resumen

La sección de Acciones y Herramientas permite configurar funcionalidades avanzadas que potencian las capacidades de tu agente conversacional durante y después de las llamadas. Incluye transferencias telefónicas, detección automática de contestadores, webhooks personalizados, protocolos de rellamada, integración de calendario y próximamente comunicación multicanal con Email, SMS y WhatsApp.

<Info>
Las funcionalidades configuradas aquí son clave para crear experiencias conversacionales fluidas y automatizaciones poderosas que integren con tus sistemas existentes, maximizando la eficiencia operacional y la satisfacción del cliente.
</Info>

## Transferencia de Llamadas

Configura cómo tu agente maneja las transferencias de llamada a agentes humanos, extensiones internas o sistemas telefónicos externos con plantillas multiidioma optimizadas.

### Tipos de Transferencia Disponibles

Meetzy soporta dos métodos principales de transferencia con configuración flexible:

<Tabs>
  <Tab title="Número de Teléfono">
    Transferir a números telefónicos tradicionales (móvil, fijo, internacional).
    
    **Formato requerido**: `+34600123456`
    
    **Casos de uso comunes**:
    - Escalar a agentes de soporte humano
    - Conectar con departamentos específicos
    - Transferir a supervisores o especialistas
    - Dirigir a líneas de emergencia
    - Routing geográfico por ubicación del cliente
  </Tab>
  <Tab title="SIP Trunk">
    Transferir a través del protocolo SIP para sistemas telefónicos empresariales.
    
    **Formato requerido**: `sip:extensión@gateway_ip:puerto`
    
    **Ejemplo**: `sip:100@78.136.102.246:5060`
    
    **Casos de uso comunes**:
    - Transferir a extensiones internas de PBX
    - Conectar con sistemas VoIP empresariales
    - Integrar con infraestructura telefónica existente
    - Enrutar a grupos de agentes específicos
    - Queue management inteligente
  </Tab>
</Tabs>

### Proceso de Configuración

<Steps>
  <Step title="Activar Transferencia">
    Habilita la opción de transferencia de llamadas usando el interruptor en la interfaz de usuario.
  </Step>
  <Step title="Seleccionar Tipo de Transferencia">
    Elige entre "Número de Teléfono" o "SIP Trunk" según tu infraestructura de comunicaciones.
  </Step>
  <Step title="Configurar Destino">
    - Para números telefónicos: Introduce el número en formato E.164 (+país + número)
    - Para SIP: Especifica la URI completa con extensión, IP y puerto
  </Step>
  <Step title="Definir Plantilla de Transferencia">
    Selecciona una plantilla predefinida optimizada para tu idioma o crea una personalizada.
  </Step>
  <Step title="Validar Configuración">
    Prueba la configuración con llamadas de test para verificar conectividad.
  </Step>
</Steps>

### Condiciones Adicionales de Finalización

El agente finalizará llamadas automáticamente en estas situaciones optimizadas:

- **Despedida Natural**: El usuario se despide claramente y el agente completa protocolarmente su despedida
- **Objetivos Completados**: Se han alcanzado exitosamente todos los objetivos específicos de la conversación
- **Timeout Configurado**: Se supera el tiempo máximo establecido para la duración de la llamada (configurable por agente)
- **Detección de Silencio Prolongado**: Períodos extendidos sin respuesta audible del interlocutor (umbral ajustable)
- **Interrupción de Línea**: Llamada cortada inesperadamente por problemas técnicos o cuelgue del cliente
- **Transferencia Exitosa**: Después de completar satisfactoriamente la transferencia a un agente humano
- **Detección de Hostilidad**: Cuando se identifica agresividad sostenida o abuso verbal del interlocutor
- **Compliance Violation**: Si se detecta solicitud de opt-out o violación de políticas de comunicación

## Herramientas Webhook Personalizadas

Crea integraciones poderosas con sistemas externos mediante webhooks que tu agente puede ejecutar durante las conversaciones en tiempo real, con configuración avanzada y monitoreo completo.

### Creando una Nueva Herramienta Webhook

<Steps>
  <Step title="Iniciar Creación">
    Haz clic en "Añadir Herramienta Webhook" para crear una nueva integración personalizada con sistemas externos.
  </Step>
  <Step title="Configuración Básica">
    - **Nombre de Herramienta**: Identificador único y descriptivo (ej. "consultar_inventario", "crear_lead_crm")
    - **Descripción Detallada**: Explica específicamente cuándo y por qué la IA debe usar esta herramienta
    - **Contexto de Uso**: Define escenarios precisos que deben activar la herramienta automáticamente
  </Step>
  <Step title="Configuración de Endpoint">
    - **URL de Destino**: Endpoint HTTP/HTTPS que procesará las peticiones
    - **Método HTTP**: Selecciona GET, POST, PUT, PATCH o DELETE según la operación requerida
    - **Encabezados**: Configura autenticación, content-type y metadatos requeridos
  </Step>
  <Step title="Definir Parámetros">
    Configure parámetros de query string y campos del cuerpo de petición con validaciones específicas.
  </Step>
  <Step title="Configuración Avanzada">
    Ajusta timeouts, comportamiento de interrupciones, modo de ejecución y feedback de audio personalizado.
  </Step>
</Steps>

### Configuración Detallada de Parámetros

Cada parámetro webhook se puede configurar con precisión técnica avanzada:

| Propiedad | Descripción | Opciones Disponibles | Ejemplos de Uso |
|----------|-------------|---------------------|-----------------|
| **Identificador** | Nombre del parámetro en la API | string único (snake_case recomendado) | customer_id, product_sku, appointment_date |
| **Tipo de Dato** | Formato y validación del valor | string, number, boolean, integer, array, object | string para emails, integer para IDs |
| **Campo Requerido** | Obligatoriedad para la ejecución | true (obligatorio) / false (opcional) | email required: true, notes optional: false |
| **Descripción para IA** | Contexto detallado que guía el uso | texto descriptivo y ejemplos de uso | "Email del cliente obtenido durante conversación" |
| **Fuente del Valor** | Origen de los datos | llm_prompt (IA genera) / static (valor fijo) | IA extrae email, valor fijo: "source": "phone_call" |
| **Valores Enum** | Restricción a opciones predefinidas | lista cerrada de valores permitidos | ["urgente", "normal", "bajo"] para prioridades |
| **Validación** | Reglas adicionales de formato | regex, rangos numéricos, longitud mín/máx | email regex, telefono 9-15 dígitos |

### Configuración Avanzada de Webhooks

#### Control de Interrupciones Durante Ejecución

<Info>
Controla si el usuario puede interrumpir al agente mientras espera la respuesta del webhook. Esta configuración es crucial para operaciones críticas que requieren completarse sin interrupciones, mejorando la fiabilidad de transacciones importantes.
</Info>

- **Deshabilitar Interrupciones**: El agente no puede ser interrumpido durante la ejecución (ideal para transacciones críticas)
- **Permitir Interrupciones**: El usuario puede interrumpir mientras se procesa (recomendado para consultas informativas rápidas)

#### Modos de Ejecución de Webhooks

| Modo | Comportamiento | Casos de Uso Ideales | Ventajas | Desventajas |
|------|---------------|---------------------|----------|-------------|
| **Inmediato** | Ejecuta durante la conversación | Consultas de datos, validaciones en tiempo real | Respuesta inmediata, UX fluida | Puede afectar fluidez si es lento |
| **Diferido** | Ejecuta después de finalizar llamada | Logs, notificaciones, actualizaciones CRM | No interrumpe conversación | Usuario no recibe confirmación directa |

#### Configuración de Feedback de Audio

- **Sonido de Herramienta**: Reproduce efecto sonoro de "tecleado" durante ejecución para proporcionar feedback auditivo al usuario
- **Control de Audio**: Automático (siempre reproduce) o inteligente (controlado por la IA según contexto conversacional)
- **Síntesis Previa Forzada**: Garantiza que el agente termine de hablar completamente antes de ejecutar la herramienta

## Protocolo de Rellamada Automática (Recall)

Configura un sistema inteligente de rellamadas automáticas cuando no se logran completar los objetivos de la conversación inicial o se requiere continuidad en la comunicación, con cumplimiento legal estricto.

### Configuración del Sistema Recall

<Steps>
  <Step title="Activar Protocolo de Rellamada">
    Habilita el sistema de rellamadas automáticas desde el panel de configuración avanzada con controles granulares.
  </Step>
  <Step title="Configurar Límites de Intentos">
    Establece el número máximo de intentos de rellamada (recomendado: 2-3 para evitar molestias y cumplir regulaciones).
  </Step>
  <Step title="Definir Estrategia de Intervalos">
    Configure intervalos progresivos entre intentos para maximizar probabilidades de éxito y minimizar intrusión.
  </Step>
  <Step title="Establecer Condiciones de Activación">
    Especifica detalladamente qué situaciones técnicas y comerciales deben desencadenar una rellamada automática.
  </Step>
  <Step title="Configurar Horarios y Compliance">
    Define ventanas horarias, zonas horarias y restricciones de días para respetar preferencias y regulaciones.
  </Step>
</Steps>

### Estrategias de Intervalos Optimizadas por Contexto

| Intervalo de Reintento | Contexto de Aplicación | Ejemplos de Uso Específicos | Tasa de Éxito Esperada |
|----------------------|----------------------|----------------------------|----------------------|
| **5-15 minutos** | Línea ocupada, urgencia alta, problemas técnicos | Confirmaciones médicas urgentes, servicios de emergencia | 65-75% |
| **30 minutos - 1 hora** | No contestó, mismo día hábil | Citas médicas del día, servicios técnicos urgentes | 50-60% |
| **2-4 horas** | Seguimiento comercial activo, horario laboral | Ventas consultivas B2B, ofertas con tiempo limitado | 40-50% |
| **Siguiente día hábil** | Seguimiento profesional estándar | Propuestas comerciales, información de productos/servicios | 30-40% |
| **48-72 horas** | Nurturing programado, leads tibios | Seguimiento de leads cualificados, ofertas especiales | 20-30% |
| **1 semana** | Mantenimiento de relación, seguimiento largo plazo | Check-ins periódicos, renovaciones, campañas estacionales | 15-25% |

### Condiciones de Activación Inteligente Detalladas

Las rellamadas se activan automáticamente en diversos escenarios técnicos y de negocio optimizados:

<Tabs>
  <Tab title="Condiciones Técnicas">
    **Situaciones de infraestructura que requieren reintento automático:**
    
    - **No Response (Ring No Answer)**: Timbre sin respuesta después de tiempo configurado (15-30 segundos)
    - **Busy Signal**: Línea ocupada o todas las líneas del destino congestionadas  
    - **Voicemail Detection**: Se detectó contestador automático mediante análisis de patrones de audio
    - **Network Issues**: Problemas de conectividad, latencia alta (>2000ms), o pérdida significativa de paquetes
    - **Call Dropped**: Llamada cortada inesperadamente por problemas técnicos del operador
    - **Poor Audio Quality**: Calidad de audio insuficiente para conversación efectiva (SNR < 15dB)
    - **Carrier Rejection**: Rechazo por parte del operador telefónico (número inválido, bloqueado)
    - **Invalid Number Format**: Número mal formateado que requiere corrección y reintento automático
    - **Timeout**: Llamada no completada dentro del tiempo límite establecido
    - **SIP Errors**: Errores específicos del protocolo SIP (503, 480, 486, etc.)
  </Tab>
  <Tab title="Condiciones de Negocio">
    **Situaciones conversacionales que justifican seguimiento estratégico:**
    
    - **Incomplete Objective**: Objetivos principales de la llamada no completados satisfactoriamente
    - **Customer Request**: Cliente solicitó explícitamente ser contactado en momento específico futuro
    - **Information Pending**: Información crítica pendiente de confirmación o recopilación adicional
    - **Follow-up Required**: El tipo de interacción requiere seguimiento según protocolo empresarial
    - **Partial Completion**: Conversación interrumpida antes de completar proceso crítico de negocio
    - **Appointment Confirmation**: Confirmación de cita programada según protocolo médico/empresarial
    - **Payment Reminder**: Recordatorio de pago con protocolo de cobranza específico y escalonado
    - **Document Verification**: Verificación de documentos o información pendiente de validación
    - **Interest Shown**: Cliente mostró interés pero requiere más tiempo para decidir
    - **Technical Issues During Call**: Problemas técnicos impidieron completar la interacción exitosamente
  </Tab>
  <Tab title="Condiciones Personalizadas">
    **Configuraciones específicas del negocio y compliance:**
    
    - **Business Hours Only**: Rellamar exclusivamente dentro de horarios comerciales configurados
    - **Customer Timezone**: Respetar automáticamente zona horaria del cliente objetivo
    - **Campaign Rules**: Reglas específicas de la campaña de marketing o playbook comercial
    - **Lead Score Threshold**: Basado en puntuación automática de calidad y potencial del contacto
    - **Previous Interaction History**: Análisis del historial completo de interacciones previas del cliente
    - **Seasonal Restrictions**: Evitar períodos vacacionales, festivos nacionales, o baja actividad comercial
    - **Channel Preference**: Priorizar canal de comunicación preferido expresado por el cliente
    - **Compliance Requirements**: Cumplir regulaciones específicas de telecomunicaciones y protección de datos
    - **Industry Specific**: Reglas particulares del sector (sanitario, financiero, legal, etc.)
    - **VIP Customer Status**: Tratamiento especial para clientes de alto valor con protocolos diferenciados
  </Tab>
</Tabs>

<Warning>
**Cumplimiento Legal Obligatorio**: El uso de protocolos de rellamada debe cumplir estrictamente con regulaciones locales e internacionales. Esto incluye respetar listas de "No Llamar", preferencias explícitas del cliente, GDPR/LOPD para protección de datos, y regulaciones específicas de telecomunicaciones como la Ley de Protección de Datos y la normativa de comunicaciones comerciales. El incumplimiento puede resultar en sanciones legales severas y daño reputacional significativo.
</Warning>

## Configuración de Caller ID para Llamadas Salientes

Gestiona los números telefónicos que aparecerán como identificador de llamada cuando tu agente realice llamadas salientes, crucial para establecer confianza, cumplir regulaciones y maximizar tasas de respuesta.

### Gestión Avanzada de Números de Caller ID

<Steps>
  <Step title="Acceder a Configuración de Caller ID">
    En la sección de configuración básica, ubica el campo "From" donde se muestran todos los caller IDs disponibles y verificados.
  </Step>
  <Step title="Añadir Nuevo Caller ID">
    Haz clic en "Añadir Caller ID" para registrar un nuevo número telefónico para llamadas salientes con verificación obligatoria.
  </Step>
  <Step title="Proceso de Verificación Automática">
    - Introduce el número telefónico en formato internacional completo (E.164)
    - Proporciona un nombre descriptivo para identificar claramente el número
    - Especifica el propósito comercial del caller ID (ventas, soporte, etc.)
    - Inicia el proceso de verificación automática del sistema
  </Step>
  <Step title="Confirmar Verificación Telefónica">
    - Recibirás una llamada automática de verificación en el número proporcionado
    - Introduce el código de verificación de 4-6 dígitos que escucharás claramente
    - Confirma la verificación para activar completamente el caller ID
    - El sistema validará la propiedad legal del número automáticamente
  </Step>
  <Step title="Asignar y Configurar">
    Una vez verificado exitosamente, selecciona el caller ID validado para tu agente conversacional específico.
  </Step>
</Steps>

### Estados Detallados del Caller ID

| Estado | Descripción | Acciones Disponibles | Tiempo Estimado |
|--------|-------------|---------------------|-----------------|
| **Verificación Pendiente** | Número añadido pero no verificado aún | Verificar, Eliminar, Reenviar código | 2-5 minutos |
| **Verificado** | Número validado y listo para usar | Usar como Caller ID, Editar nombre, Eliminar | Inmediato |
| **Verificación Fallida** | Error en el proceso de verificación | Reintentar Verificación, Eliminar, Soporte | 24 horas cooldown |
| **En Uso** | Actualmente asignado a uno o más agentes | Ver Detalles, Cambiar asignación, Estadísticas | Inmediato |
| **Suspendido** | Temporalmente deshabilitado por compliance | Revisar motivo, Solicitar reactivación | Variable |
| **Bloqueado** | Bloqueado por operadores o compliance | Contactar soporte, Revisar políticas | Requiere revisión |

### Consideraciones Legales y Técnicas Avanzadas

<Warning>
**Compliance Caller ID Estricto**: Solo puedes usar números telefónicos que poseas legalmente o tengas autorización explícita y documentada para usar. El spoofing de caller ID sin autorización es ilegal en la mayoría de jurisdicciones y puede resultar en sanciones severas, multas significativas y daño reputacional permanente.
</Warning>

#### Requisitos de Compliance

- **Verificación Obligatoria**: Todos los caller IDs deben ser verificados mediante llamada automática y validación de propiedad
- **Números Propios**: Solo registra números telefónicos que poseas legalmente o controles bajo contrato válido
- **Regulaciones Locales**: Cumple con regulaciones específicas de caller ID en tu jurisdicción (TCPA, GDPR, etc.)
- **Transparencia Empresarial**: Usa caller IDs que identifiquen claramente tu empresa u organización
- **Documentación Legal**: Mantén documentación que demuestre la propiedad o autorización legal del número

#### Mejores Prácticas para Caller ID

- **Números Locales**: Usa números locales del área geográfica de tus clientes para mayor confianza
- **Consistencia de Marca**: Mantén consistencia en los caller IDs usados para el mismo tipo de comunicaciones
- **Monitoreo de Reputación**: Supervisa la reputación de tus números ante operadores telefónicos
- **Rotación Inteligente**: Alterna números para evitar saturación y filtros de spam
- **Análisis de Performance**: Analiza tasas de respuesta por caller ID para optimización continua

## Solución de Problemas Avanzada y Debugging

<Tabs>
  <Tab title="Problemas de Transferencia">
    **Síntoma**: Las transferencias fallan constantemente o no se conectan exitosamente
    
    **Diagnóstico paso a paso:**
    ```bash
    # 1. Verificar conectividad del número destino
    curl -X POST "https://api.meetzy.com/test-call" \
      -H "Content-Type: application/json" \
      -H "Authorization: Bearer YOUR_API_KEY" \
      -d '{"number": "+34600123456", "test_mode": true}'
    
    # 2. Para SIP, verificar conectividad del gateway
    telnet your-sip-gateway.com 5060
    nmap -p 5060 your-sip-gateway.com
    
    # 3. Verificar formato de número (debe ser E.164)
    echo "+34600123456" | grep -E '^\+[1-9]\d{1,14}$'
    
    # 4. Test de latencia y calidad de conexión
    ping -c 10 your-sip-gateway.com
    traceroute your-sip-gateway.com
    ```
    
    **Soluciones comunes por orden de frecuencia:**
    - **Formato incorrecto (40%)**: Usar formato E.164 (+país + número sin espacios ni caracteres especiales)
    - **Permisos insuficientes (25%)**: Verificar permisos de transferencia en la cuenta y carrier
    - **SIP mal configurado (15%)**: Validar URI SIP, IP correcta y puerto accesible
    - **Firewall bloqueando (10%)**: Abrir puertos SIP (5060) y RTP (10000-20000) en ambas direcciones
    - **Credenciales erróneas (5%)**: Verificar usuario/password SIP en gateway
