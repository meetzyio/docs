## ¿Qué es Copilot?

Copilot es un editor de prompts impulsado por IA integrado en el Editor Avanzado. En lugar de solo darte sugerencias, **Copilot edita y mejora directamente el system prompt de tu agente** basándose en tus instrucciones. Analiza tu prompt actual, genera mejoras y aplica los cambios automáticamente.

<Info>
Copilot no solo sugiere: edita. Describe lo que quieres mejorar y observa cómo modifica tu system prompt en tiempo real usando un sistema de eventos avanzado SSE (Server-Sent Events).
</Info>

## Accediendo a Copilot

Copilot está ubicado en el panel derecho del Editor Avanzado. Haz clic en la pestaña "Copilot" para acceder, o usa el atajo de teclado `⌘J`.

## El Orbe de Copilot: la cara de Meetzy

Copilot tiene una identidad visual propia: un **orbe animado con degradado** que actúa como la "cara" de Meetzy en toda la experiencia. Este orbe reemplaza los íconos genéricos de chat y aparece en tres lugares clave:

- **Pantalla de bienvenida**: el orbe da la bienvenida con una animación de entrada suave (`hero-fade-up`).
- **Avatar de cada mensaje**: cada respuesta de Copilot lleva el orbe como avatar.
- **Indicador de escritura**: mientras Copilot procesa tu solicitud, el orbe cambia de estado.

<Tabs>
  <Tab title="Estado Idle (reposo)">
    Cuando Copilot no está procesando nada, el orbe respira con un pulso lento y ambiental, transmitiendo calma y disponibilidad.

    **Uso típico**: pantalla de bienvenida, avatares de mensajes ya completados.
  </Tab>
  <Tab title="Estado Active (activo)">
    Mientras Copilot está "pensando" o transmitiendo una respuesta, el orbe acelera su pulso y despliega **anillos expansivos** (con un ligero retraso entre ellos), evocando visualmente una llamada conectándose — un guiño directo a que Meetzy es un copiloto de agentes de voz, no un chatbot genérico.

    **Uso típico**: mensajes con transmisión activa (`isStreaming`), indicador de "Pensando".
  </Tab>
</Tabs>

El componente admite tres tamaños (`sm`, `md`, `lg`) según el contexto: pequeño para avatares en línea dentro de la conversación, y grande para la pantalla de bienvenida.

<Info>
El orbe no es solo decorativo: su estado (`idle` / `active`) está sincronizado en tiempo real con el ciclo de vida de cada mensaje, así que siempre sabes de un vistazo si Copilot sigue trabajando.
</Info>

## Pantalla de Bienvenida

Cuando abres Copilot por primera vez, verás al orbe dando la bienvenida junto a un mensaje corto ("Vamos a afinar este agente") y **cuatro tarjetas de acción rápida** con bloques de color sólido, pensadas para cubrir el ciclo completo de mejora de un agente: diagnosticar, encontrar fallos, crear una salvaguarda y verificar la mejora.

<Steps>
  <Step title="Diagnosticar rendimiento (violeta)">
    Un clic para que Copilot analice el rendimiento del agente usando métricas reales, evaluaciones fallidas y llamadas relevantes, mostrando los hallazgos clave de forma visual.

    **Prompt enviado**: "Diagnostica el rendimiento de este agente usando métricas reales, evaluaciones fallidas y llamadas relevantes. Muestra los hallazgos clave de forma visual."
  </Step>
  <Step title="Encontrar fallos recurrentes (rosa)">
    Identifica patrones de fallo repetidos en el agente y los presenta en una tabla con ejemplos extraídos de llamadas reales.

    **Prompt enviado**: "Encuentra fallos recurrentes en este agente y muéstralos en una tabla con ejemplos de llamadas reales."
  </Step>
  <Step title="Crear un test de regresión (verde)">
    Convierte una llamada real que falló en un test automatizado, para que ese problema específico no vuelva a ocurrir.

    **Prompt enviado**: "Crea un test de regresión a partir de una llamada real que falló, para que este problema no vuelva a ocurrir."
  </Step>
  <Step title="Mejorar y verificar (ámbar)">
    Mejora el system prompt del agente basándose en el diagnóstico actual y sugiere cómo verificar que el cambio realmente funcionó, cerrando el ciclo de mejora.

    **Prompt enviado**: "Mejora el prompt del agente basándote en el diagnóstico actual y luego sugiere cómo verificar el cambio."
  </Step>
</Steps>

<Info>
Cada tarjeta tiene su propio color de fondo, ícono y una pequeña flecha que se anima al pasar el cursor, invitando a la acción con un solo clic. No necesitas escribir nada: el prompt correspondiente se envía automáticamente a Copilot.
</Info>

## Sistema de Eventos SSE Avanzado

Copilot utiliza un protocolo de eventos Server-Sent Events (SSE) para comunicación en tiempo real entre el frontend y el backend, proporcionando feedback instantáneo durante el procesamiento.

### Tipos de Eventos SSE

<Tabs>
  <Tab title="Eventos de Procesamiento">
    **Thinking**: Análisis en tiempo real del prompt
    ```json
    {
      "type": "thinking",
      "seconds": 3
    }
    ```
    - Muestra progreso del análisis cognitivo
    - Duración típica: 2-8 segundos
    - Incluye timer visual en la interfaz
    
    **Text**: Transmisión de contenido de respuesta
    ```json
    {
      "type": "text",
      "content": "Analizando el system prompt actual..."
    }
    ```
    - Explicaciones paso a paso del proceso
    - Razonamiento transparente de las decisiones
    - Contexto para mejoras propuestas
  </Tab>
  <Tab title="Eventos de Edición">
    **Edit**: Modificaciones estructuradas del prompt
    ```json
    {
      "type": "edit",
      "operation": "replace_section",
      "targetSection": "Greeting Instructions",
      "newContent": "Say hello warmly and...",
      "summary": "Improved greeting warmth",
      "linesAdded": 3
    }
    ```
    
    **Operaciones disponibles**:
    - `replace_section`: Reemplaza sección completa
    - `append_section`: Añade nueva sección al final
    - `insert_after`: Inserta después de una sección
    - `delete_section`: Elimina sección existente
    
    **Complete**: Finalización de la operación
    ```json
    {
      "type": "complete",
      "duration": "2.3s",
      "editsCount": 2,
      "edits": [
        {
          "operation": "replace_section",
          "targetSection": "Greeting Instructions",
          "summary": "Enhanced greeting protocol"
        }
      ]
    }
    ```
  </Tab>
  <Tab title="Eventos de Configuración">
    **Input_Change**: Modificación de parámetros de entrada
    ```json
    {
      "type": "input_change",
      "name": "customer_type",
      "fieldType": "string",
      "description": "Type of customer (new, returning, vip)",
      "required": false
    }
    ```
    
    **Output_Change**: Campos de salida automáticos
    ```json
    {
      "type": "output_change",
      "name": "interest_level",
      "fieldType": "number",
      "description": "Customer interest level (1-10)"
    }
    ```
    
    **Evaluation_Change**: Creación de evaluaciones
    ```json
    {
      "type": "evaluation_change",
      "name": "objection_handling",
      "prompt": "Rate how well the agent handles price objections"
    }
    ```
    
    **Agent_Config**: Configuración global del agente
    ```json
    {
      "type": "agent_config",
      "greeting_inbound_prompt": "Updated inbound greeting",
      "greeting_outbound_prompt": "Updated outbound greeting",
      "phone_number": "+1234567890",
      "phone_sid": "PN123..."
    }
    ```

    **Audio_Tags_Set**: Ajuste de tags de audio expresivos (Eleven v3)
    ```json
    {
      "type": "audio_tags_set",
      "tags": [
        { "tag": "[warmly]", "description": "Tono cálido para el saludo" },
        { "tag": "[pause]", "description": "Pausa breve antes de confirmar datos" }
      ]
    }
    ```
  </Tab>
  <Tab title="Eventos Avanzados">
    **Test_Create**: Generación automática de tests
    ```json
    {
      "type": "test_create",
      "testType": "scenario",
      "chat_history": [
        {"role": "user", "content": "I'm interested in your services"},
        {"role": "agent", "content": "Great! Let me tell you about..."}
      ],
      "success_condition": "Agent successfully captures contact information",
      "dynamic_variables": {"customer_name": "John Doe"}
    }
    ```

    **Test_Update**: Actualización de un test ya existente
    ```json
    {
      "type": "test_update",
      "test_id": "test_8f21ac",
      "success_condition": "Agent confirms email format before saving it",
      "dynamic_variables": {"customer_name": "Jane Doe"}
    }
    ```
    - Modifica un test previamente creado sin necesidad de recrearlo desde cero
    - Solo se envían los campos que cambian; el resto del test se conserva intacto
    
    **Enrichment_Set**: Configuración de output enrichment
    ```json
    {
      "type": "enrichment_set",
      "content": "Additional context for output processing..."
    }
    ```
    
    **Rich_Block**: Bloque de contenido visual enriquecido
    ```json
    {
      "type": "rich_block",
      "blockType": "stat_grid",
      "data": {
        "title": "Resumen de rendimiento",
        "stats": [
          { "label": "Tasa de éxito", "value": "78%", "trend": "up", "tone": "success" }
        ]
      }
    }
    ```
    - Permite a Copilot responder con tarjetas, tablas, gráficos o grids de estadísticas en lugar de solo texto
    
    **Error**: Manejo de errores durante procesamiento
    ```json
    {
      "type": "error",
      "message": "Unable to process request due to network timeout"
    }
    ```
  </Tab>
</Tabs>

### Flujo de Procesamiento SSE Detallado

<Steps>
  <Step title="Inicialización de Conexión">
    - Copilot establece conexión SSE bidireccional
    - Se configura contexto del prompt actual
    - Validación de permisos y recursos disponibles
    - Preparación del controlador de cancelación (`AbortController`)
  </Step>
  <Step title="Análisis Cognitivo (Thinking)">
    - **Duración**: 2-8 segundos según complejidad
    - **Indicadores**: Timer visual con segundos transcurridos
    - **Procesos**: 
      - Parsing de la estructura actual del prompt
      - Identificación de secciones y marcadores
      - Análisis de contexto de llamadas si se referencian
      - Planificación de estrategia de mejoras
  </Step>
  <Step title="Generación de Contenido (Text)">
    - **Streaming**: Contenido transmitido en tiempo real
    - **Transparencia**: Explicación del razonamiento detrás de los cambios
    - **Contextualización**: Referencias específicas a problemas identificados
    - **Preparación**: Configuración de operaciones de edición estructurada
  </Step>
  <Step title="Aplicación de Cambios (Edit)">
    - **Precisión**: Operaciones granulares sobre secciones específicas
    - **Atomicidad**: Cada operación es independiente y reversible
    - **Seguimiento**: Métricas detalladas de líneas afectadas
    - **Validación**: Verificación de integridad estructural
  </Step>
  <Step title="Finalización y Sugerencias (Complete)">
    - **Resumen**: Estadísticas completas de la operación
    - **Duración**: Tiempo total de procesamiento
    - **Métricas**: Número de ediciones y líneas modificadas
    - **Sugerencias**: Próximos pasos inteligentes automáticos
  </Step>
</Steps>

## Bloques Enriquecidos en las Respuestas

Además de texto y ediciones, Copilot puede responder con **bloques visuales enriquecidos** cuando el análisis se beneficia de una representación estructurada. Estos bloques se insertan directamente en el flujo de la conversación, mezclados con el texto normal.

<Info>
Un mismo mensaje de Copilot puede combinar texto explicativo con uno o varios bloques enriquecidos, por ejemplo: un párrafo de diagnóstico seguido de un `stat_grid` con las métricas clave y una `table` con ejemplos de llamadas problemáticas.
</Info>

### Tipos de Bloques Disponibles

<Tabs>
  <Tab title="Stat Grid (grid de estadísticas)">
    Ideal para mostrar KPIs de un vistazo: tasa de éxito, duración promedio, número de llamadas analizadas, etc.

    ```json
    {
      "blockType": "stat_grid",
      "data": {
        "title": "Rendimiento últimos 30 días",
        "stats": [
          {
            "label": "Tasa de éxito",
            "value": "82%",
            "delta": "+5%",
            "trend": "up",
            "tone": "success",
            "hint": "vs. mes anterior"
          },
          {
            "label": "Duración promedio",
            "value": "6.4 min",
            "delta": "-1.2 min",
            "trend": "down",
            "tone": "info"
          }
        ]
      }
    }
    ```

    **Campos soportados por cada estadística**:
    - `label`: Nombre de la métrica
    - `value`: Valor a mostrar (texto, número o booleano)
    - `delta`: Variación respecto a un período anterior (opcional)
    - `trend`: `up`, `down` o `flat` — determina el ícono de tendencia
    - `tone`: `neutral`, `info`, `success`, `warning` o `danger` — define el color de la tarjeta
    - `hint`: Texto secundario aclaratorio
  </Tab>
  <Tab title="Table (tabla)">
    Perfecta para listar ejemplos de llamadas, resultados de tests o comparaciones fila por fila.

    ```json
    {
      "blockType": "table",
      "data": {
        "title": "Llamadas con fallo en captura de email",
        "columns": [
          { "key": "call_id", "label": "Llamada" },
          { "key": "issue", "label": "Problema" },
          { "key": "status", "label": "Estado" }
        ],
        "rows": [
          { "call_id": "abc123", "issue": "Email mal transcrito", "status": "failed" },
          { "call_id": "def456", "issue": "Cliente colgó antes de confirmar", "status": "pending" }
        ],
        "footnote": "Basado en 12 llamadas analizadas"
      }
    }
    ```

    **Características**:
    - Encabezado fijo (`sticky`) al hacer scroll dentro de tablas largas
    - Valores como `"si"`, `"passed"`, `"success"` se resaltan automáticamente en verde
    - Valores como `"no"`, `"failed"`, `"error"` se resaltan en rojo
    - Valores como `"pending"`, `"still_running"` se resaltan en ámbar
    - Soporta una nota al pie (`footnote`) opcional
  </Tab>
  <Tab title="Chart (gráfico)">
    Genera visualizaciones con Chart.js directamente dentro del chat, útiles para mostrar tendencias a lo largo del tiempo.

    ```json
    {
      "blockType": "chart",
      "data": {
        "title": "Evolución de satisfacción del cliente",
        "chart_type": "line",
        "labels": ["Semana 1", "Semana 2", "Semana 3", "Semana 4"],
        "datasets": [
          { "label": "Satisfacción (1-10)", "data": [6.2, 6.8, 7.1, 7.9] }
        ]
      }
    }
    ```

    **Tipos de gráfico soportados** (`chart_type`):
    - `line`: Tendencias a lo largo del tiempo
    - `bar`: Comparaciones entre categorías
    - `pie` / `doughnut`: Distribuciones y proporciones

    Cada dataset se colorea automáticamente con una paleta consistente y se adapta al modo oscuro.
  </Tab>
  <Tab title="Card (tarjeta)">
    Bloques de texto destacado con título, cuerpo y lista de puntos, ideales para resúmenes o recomendaciones puntuales.

    ```json
    {
      "blockType": "card",
      "data": {
        "title": "Recomendación",
        "body": "Se detectó que el agente no confirma el email en el 40% de los casos.",
        "bullets": [
          "Añadir paso de confirmación explícita",
          "Validar formato antes de guardar",
          "Repetir el email en voz alta al cliente"
        ],
        "tone": "warning"
      }
    }
    ```

    **Tonos disponibles** (`tone`): `neutral`, `info`, `success`, `warning`, `danger` — cada uno con su propia paleta de color adaptada a modo claro y oscuro.
  </Tab>
</Tabs>

### Cuándo Usa Copilot Cada Bloque

| Bloque | Se usa cuando... |
|--------|-------------------|
| `stat_grid` | Necesitas comparar varias métricas numéricas de un vistazo (KPIs, tasas, promedios) |
| `table` | Hay múltiples ejemplos o filas de datos que comparar (llamadas, tests, errores) |
| `chart` | La información tiene una dimensión temporal o de distribución que se entiende mejor visualmente |
| `card` | Copilot quiere destacar una recomendación, hallazgo o advertencia puntual |

<Info>
Todos los bloques enriquecidos son totalmente responsivos y se adaptan automáticamente al modo claro/oscuro del editor, manteniendo consistencia visual con el resto de la interfaz.
</Info>

## Edición Directa de Tags de Audio y Actualización de Tests

Además de reescribir el system prompt, Copilot puede actuar directamente sobre dos elementos clave de la configuración del agente sin pedirte que los reconstruyas manualmente: los **tags de audio expresivos** y los **tests ya existentes**.

<Tabs>
  <Tab title="Modificación de Tags de Audio">
    Los agentes de voz sobre motores como Eleven v3 admiten tags expresivos (por ejemplo `[warmly]`, `[pause]`, `[excited]`) que modulan cómo se pronuncia el texto. Copilot puede añadir, ajustar o eliminar estos tags directamente sobre la configuración del agente cuando detecta que mejorarían la naturalidad de la voz.

    ```json
    {
      "type": "audio_tags_set",
      "tags": [
        { "tag": "[warmly]", "description": "Tono cálido para el saludo inicial" },
        { "tag": "[pause]", "description": "Pausa breve antes de confirmar el email" }
      ]
    }
    ```

    **Cuándo lo hace Copilot**:
    - Al diagnosticar que el agente suena demasiado plano o robótico en ciertas partes
    - Cuando el prompt describe una emoción o énfasis que no está reflejado en tags reales
    - Como parte de una mejora de "Mejorar y verificar" enfocada en tono de voz

    <Info>
    Los cambios de tags se muestran como un evento independiente en el chat, con una vista previa de los tags antes y después, para que puedas aceptarlos o descartarlos igual que una edición de prompt.
    </Info>
  </Tab>
  <Tab title="Actualización de Tests Existentes">
    Antes, Copilot solo podía **crear** tests nuevos a partir de llamadas reales. Ahora también puede **actualizar un test que ya existe**, sin duplicarlo ni perder su historial de ejecuciones.

    ```json
    {
      "type": "test_update",
      "test_id": "test_8f21ac",
      "success_condition": "Agent confirms email format before saving it",
      "dynamic_variables": { "customer_name": "Jane Doe" }
    }
    ```

    **Comportamiento clave**:
    - Solo se modifican los campos incluidos en el evento; el resto del test permanece igual
    - El `test_id` identifica de forma inequívoca qué test se está actualizando
    - Es ideal cuando un test dejó de reflejar el comportamiento esperado tras una mejora del prompt

    **Ejemplo de flujo típico**:
    1. Copilot detecta que un test de regresión ya no representa el escenario real (el bug cambió ligeramente)
    2. En vez de crear un test duplicado, emite un evento `test_update` sobre el `test_id` original
    3. El test conserva su nombre e historial, pero con la condición de éxito actualizada
  </Tab>
</Tabs>

<Warning>
Tanto los cambios de tags de audio como las actualizaciones de tests requieren tu confirmación antes de aplicarse de forma permanente, igual que cualquier otra edición propuesta por Copilot.
</Warning>

## Operaciones de Edición Estructurada Avanzada

Copilot utiliza un sistema sofisticado de operaciones para modificar el prompt de manera precisa, controlada y rastreable.

### Arquitectura de Marcadores de Sección

Los marcadores permiten identificación y manipulación precisa de secciones:

```
# === GREETING INSTRUCTIONS ===
[Contenido de saludo personalizable]

# === OBJECTION HANDLING ===
[Estrategias específicas de manejo]

# === OUTPUT REQUIREMENTS ===
[Campos y formatos de salida]

# === EVALUATION CRITERIA ===
[Métricas de éxito definidas]
```

<Info>
Los marcadores son insertados automáticamente por Copilot y permiten operaciones quirúrgicas sin afectar otras partes del prompt.
</Info>

### Tipos de Operaciones con Casos de Uso

| Operación | Descripción Técnica | Casos de Uso | Impacto |
|-----------|---------------------|--------------|---------|
| `replace_section` | Reemplaza contenido completo entre marcadores | Refactor de lógica existente, mejoras de tono | Alto |
| `append_section` | Añade nueva sección al final del prompt | Nuevas funcionalidades, expansión de capacidades | Medio |
| `insert_after` | Inserta contenido después de sección específica | Contexto adicional, validaciones intermedias | Bajo |
| `delete_section` | Elimina sección y sus marcadores | Limpieza de código obsoleto, simplificación | Medio |
| `modify_inline` | Cambios granulares dentro de líneas | Ajustes de tono, correcciones específicas | Bajo |

### Ejemplo de Operación Completa

```json
{
  "type": "edit",
  "operation": "replace_section",
  "targetSection": "Email Capture Protocol",
  "newContent": "When collecting email addresses:\n1. Ask clearly and naturally\n2. Repeat back for confirmation\n3. Validate format if possible\n4. Save to 'customer_email' field",
  "summary": "Enhanced email capture with validation steps",
  "linesAdded": 4,
  "metadata": {
    "previousLines": 2,
    "improvement": "Added confirmation and validation steps",
    "rationale": "Based on @call:abc123 where email was incorrectly captured"
  }
}
```

## Sistema de Grabación de Voz Avanzado

Copilot incluye capacidades de grabación de voz sofisticadas para interacción natural y sin usar las manos.

### Modos de Grabación Disponibles

<Tabs>
  <Tab title="Grabación Manual por Botón">
    **Activación**: Clic en ícono de micrófono
    
    **Características**:
    - Control total sobre inicio/pausa/parada
    - Ideal para solicitudes largas y complejas
    - Permite revisión antes de envío
    - Indicadores visuales de estado de grabación
    
    **Flujo de trabajo**:
    1. Clic en micrófono → Solicitar permisos
    2. Grabación activa con feedback visual
    3. Clic en "Parar" → Procesamiento automático
    4. Transcripción y envío a Copilot
  </Tab>
  <Tab title="Mantener presionado (tecla Alt)">
    **Activación**: Mantener presionada la tecla Alt
    
    **Características**:
    - Estilo "walkie-talkie" para rapidez
    - Ideal para correcciones y ajustes rápidos
    - Se detiene automáticamente al soltar la tecla
    - Feedback háptico en dispositivos compatibles
    
    **Casos de uso**:
    - Refinamientos rápidos: "Hazlo menos formal"
    - Correcciones inmediatas: "El tono es muy agresivo"
    - Instrucciones puntuales: "Añade validación de email"
  </Tab>
  <Tab title="Detección de Silencio Inteligente">
    **Activación**: Automática durante grabación
    
    **Características**:
    - Detección de pausas naturales
    - Finalización automática tras silencio prolongado
    - Algoritmos de puerta de ruido (noise gate) para filtrado
    - Adaptación a patrones de habla individuales
    
    **Configuración**:
    - Umbral de silencio: 2-5 segundos
    - Sensibilidad ajustable según ambiente
    - Cancelación de ruido de fondo
  </Tab>
</Tabs>

### Implementación Técnica de Voz

<Steps>
  <Step title="Inicialización del MediaRecorder">
    ```javascript
    const mediaRecorder = new MediaRecorder(stream, {
      mimeType: 'audio/webm;codecs=opus',
      audioBitsPerSecond: 128000
    });
    ```
    - Calidad optimizada para reconocimiento de voz
    - Compresión eficiente para transmisión rápida
    - Compatibilidad multi-navegador garantizada
  </Step>
  <Step title="Procesamiento en Tiempo Real">
    - **Segmentación**: Audio dividido en fragmentos de 1-2 segundos
    - **Streaming**: Procesamiento progresivo durante grabación
    - **Gestión de buffer**: Optimización de memoria y latencia
    - **Manejo de errores**: Recuperación automática de fallos de red
  </Step>
  <Step title="Transcripción vía Web Speech API">
    - **Modelo**: Optimizado para terminología técnica de AI/ML
    - **Idiomas**: Soporte para español, inglés y variantes regionales
    - **Confianza**: Puntuación de precisión para validación automática
    - **Respaldo**: Servicios en la nube para casos complejos
  </Step>
  <Step title="Post-procesamiento Inteligente">
    - **Limpieza**: Filtrado de muletillas y repeticiones
    - **Contextualización**: Interpretación de referencias como "@call:ID"
    - **Formateo**: Estructura automática para procesamiento por Copilot
    - **Validación**: Verificación de coherencia antes de envío
  </Step>
</Steps>

### Permisos y Solución de Problemas de Audio

<Warning>
La grabación de voz requiere permisos explícitos del navegador y conexión HTTPS para funcionamiento seguro.
</Warning>

| Problema | Causa Probable | Solución |
|----------|----------------|----------|
| "Micrófono no disponible" | Permisos denegados | Permitir en configuración del navegador |
| "Audio distorsionado" | Micrófono de baja calidad | Usar headset o micrófono externo |
| "Transcripción incorrecta" | Ruido de fondo excesivo | Ambiente silencioso o cancelación de ruido |
| "Grabación se corta" | Límite de tiempo excedido | Dividir en solicitudes más cortas |
