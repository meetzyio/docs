La edición colaborativa de Meetzy permite que múltiples miembros del equipo trabajen simultáneamente en el mismo agente, con un sistema de control de versiones robusto que previene conflictos y mantiene un historial completo de cambios.

## Cómo Funciona

<Info>
El sistema de colaboración utiliza conceptos similares a Git para manejar cambios locales, remotos y resolver conflictos automáticamente.
</Info>

### Estados de Colaboración

El editor muestra diferentes estados que indican el estado actual de colaboración:

- **Clean**: No hay cambios locales ni remotos pendientes
- **Modified**: Tienes cambios locales sin publicar
- **Conflict**: Hay conflictos que requieren resolución manual
- **Syncing**: Sincronizando cambios con el servidor
- **Locked**: Otro usuario está editando activamente

### Indicadores Visuales

El header del editor muestra información de estado en tiempo real:


## Flujo de Trabajo Colaborativo

<Steps>
<Step title="Realizar Cambios Localmente">
Edita cualquier campo del agente. Los cambios se rastrean automáticamente y se marcan como "modificados".

```tsx
// Cambios rastreados automáticamente
private trackChange(field: string, value: any): void {
  this.localChanges.set(field, value);
  this.updateGitStatus();
}
```
</Step>

<Step title="Verificar Actualizaciones Remotas">
Usa el botón "Check" para verificar si otros usuarios han realizado cambios.

```tsx
async checkForUpdates(): Promise<void> {
  this.gitStatus = 'syncing';
  const remoteVersion = await this.fetchRemoteVersion();
  
  if (remoteVersion > this.localVersion) {
    this.hasRemoteChanges = true;
    this.showConflictResolution();
  }
}
```
</Step>

<Step title="Sincronizar Cambios">
Utiliza "Pull" para obtener cambios remotos o "Commit" para publicar tus cambios locales.
</Step>

<Step title="Resolver Conflictos">
Si hay conflictos, el sistema te guiará a través del proceso de resolución.

```typescript
async validateSemanticCoherence(changes: Map<string, any>): Promise<SemanticValidation[]> {
  const validations: SemanticValidation[] = [];

  if (changes.has('instructions') && changes.has('greeting_inbound_prompt')) {

    const conflicts = await this.detectSemanticConflicts(instructions, greeting);

        field: 'greeting_inbound_prompt',
        content: greeting,
        conflicts,
        suggestions: await this.generateCoherenceSuggestions(instructions, greeting)
      });
    }
  }
  
  return validations;
}
```

## Sincronización Inteligente

### Merge Automático Contextual

El sistema ahora puede fusionar automáticamente cambios compatibles usando contexto semántico:

```typescript
interface IntelligentMerge {
  canAutoMerge: boolean;
  mergeStrategy: 'append' | 'prepend' | 'interleave' | 'semantic_merge';
  mergedContent: string;
  confidence: number; // 0-1
  humanReviewRequired: boolean;
}

class SmartMergeEngine {
  async attemptIntelligentMerge(
    baseValue: string,
    localValue: string, 
    remoteValue: string,
    fieldType: 'instructions' | 'prompt' | 'schema'
  ): Promise<IntelligentMerge> {
    
    // Análisis contextual del contenido
    const localChanges = await this.extractSemanticChanges(baseValue, localValue);
    const remoteChanges = await this.extractSemanticChanges(baseValue, remoteValue);
    
    // Verificar compatibilidad
    const compatibility = await this.checkChangeCompatibility(localChanges, remoteChanges);
    
    if (compatibility.score > 0.8) {
      // Intentar merge automático
      const mergedContent = await this.performSemanticMerge(
        baseValue, 
        localChanges, 
        remoteChanges,
        fieldType
      );
      
      return {
        canAutoMerge: true,
        mergeStrategy: compatibility.recommendedStrategy,
        mergedContent,
        confidence: compatibility.score,
        humanReviewRequired: compatibility.score < 0.95
      };
    }
    
    return {
      canAutoMerge: false,
      mergeStrategy: 'manual',
      mergedContent: baseValue,
      confidence: 0,
      humanReviewRequired: true
    };
  }
}
```

### Propagación de Cambios Relacionados

<Info>
Como la segunda ley de la termodinámica: la información sobre cambios debe propagarse uniformemente a través de campos relacionados.
</Info>

```typescript
interface ChangePropagatIon {
  sourceChange: FieldChange;
  propagatedChanges: PropagatedChange[];
  propagationRules: PropagationRule[];
}

interface PropagatedChange {
  targetField: string;
  suggestedValue: any;
  reason: string;
  confidence: number;
  autoApply: boolean;
}

class ChangePropagationEngine {
  async propagateChange(change: FieldChange): Promise<ChangePropagatIon> {
    const propagationRules = this.getApplicableRules(change.field);
    const propagatedChanges: PropagatedChange[] = [];
    
    for (const rule of propagationRules) {
      const suggestion = await this.generatePropagationSuggestion(change, rule);
      if (suggestion.confidence > 0.7) {
        propagatedChanges.push(suggestion);
      }
    }
    
    return {
      sourceChange: change,
      propagatedChanges,
      propagationRules
    };
  }
  
  // Ejemplo: cambios en input_schema deben reflejarse en instructions
  private getApplicableRules(field: string): PropagationRule[] {
    const rules: Record<string, PropagationRule[]> = {
      'input_schema': [
        {
          targetField: 'instructions',
          type: 'parameter_update',
          description: 'Actualizar instrucciones para reflejar nuevos parámetros',
          autoApply: false
        }
      ],
      'tools': [
        {
          targetField: 'instructions',
          type: 'tool_integration',
          description: 'Incorporar nuevas herramientas en las instrucciones',
          autoApply: false
        }
      ]
    };
    
    return rules[field] || [];
  }
}
```

### Validación Post-Resolución

<Warning>
Después de resolver conflictos, se ejecuta automáticamente una validación integral para asegurar la coherencia del agente.
</Warning>

```typescript
class PostResolutionValidator {
  async validateResolution(resolvedAssistant: Assistant): Promise<ValidationResult> {
    const validations = await Promise.all([
      this.validateSemanticCoherence(resolvedAssistant),
      this.validateStructuralIntegrity(resolvedAssistant),
      this.validateFunctionalCompatibility(resolvedAssistant),
      this.validatePromptConsistency(resolvedAssistant)
    ]);
    
    return {
      isValid: validations.every(v => v.isValid),
      errors: validations.flatMap(v => v.errors),
      warnings: validations.flatMap(v => v.warnings),
      suggestions: validations.flatMap(v => v.suggestions),
      autoFixAvailable: validations.some(v => v.autoFixAvailable)
    };
  }
  
  private async validateSemanticCoherence(assistant: Assistant): Promise<ValidationResult> {
    // Validar que instructions, prompts y schema sean coherentes
    const coherenceIssues = await this.analyzeSemanticCoherence({
      instructions: assistant.instructions,
      greeting_inbound: assistant.greeting_inbound_prompt,
      greeting_outbound: assistant.greeting_outbound_prompt,
      input_schema: assistant.input_schema,
      output_schema: assistant.output_schema
    });
    
    return {
      isValid: coherenceIssues.length === 0,
      errors: coherenceIssues.filter(i => i.severity === 'critical'),
      warnings: coherenceIssues.filter(i => i.severity === 'warning'),
      suggestions: coherenceIssues.map(i => i.suggestion),
      autoFixAvailable: coherenceIssues.some(i => i.autoFixable)
    };
  }
}
```

```typescript
interface SemanticVersion {
  major: number; // Cambios incompatibles (ej: schema breaking changes)
  minor: number; // Nueva funcionalidad compatible (ej: nuevas tools)
  patch: number; // Correcciones y mejoras menores
}

interface VersionedChange {
  version: SemanticVersion;
  changeType: 'major' | 'minor' | 'patch';
  description: string;
  affectedFields: string[];
  breakingChanges: string[];
  migrationRequired: boolean;
}

function determineVersionBump(changes: Map<string, any>): VersionedChange {
  let changeType: 'major' | 'minor' | 'patch' = 'patch';
  const affectedFields = Array.from(changes.keys());
  const breakingChanges: string[] = [];
  
  // Cambios major: modificaciones en input_schema que rompen compatibilidad
  if (changes.has('input_schema')) {
    const schemaChange = analyzeSchemaChange(changes.get('input_schema'));
    if (schemaChange.isBreaking) {
      changeType = 'major';
      breakingChanges.push('Input schema modificado de forma incompatible');
    }
  }
  
  // Cambios minor: nuevas tools o funcionalidades
  if (changes.has('tools') && changeType !== 'major') {
    const toolsChange = analyzeToolsChange(changes.get('tools'));
    if (toolsChange.hasNewTools) {
      changeType = 'minor';
    }
  }
  
  return {
    version: calculateNewVersion(getCurrentVersion(), changeType),
    changeType,
    description: generateChangeDescription(changes),
    affectedFields,
    breakingChanges,
    migrationRequired: changeType === 'major'
  };
}
```
