<Info>
El panel de consumo Billy proporciona una vista integral de tu consumo actual de minutos facturables, permitiendo el seguimiento y análisis detallado de los costos por asistente.
</Info>

## Descripción General

El panel de consumo Billy es un componente flotante que se muestra en la esquina inferior izquierda del editor avanzado. Ofrece:

- **Seguimiento en tiempo real** del consumo de minutos
- **Desglose detallado** por asistente
- **Filtros flexibles** por período
- **Gráficos históricos** para administradores
- **Indicadores visuales** del estado del servicio
- **Panel expandible/colapsible** para optimizar el espacio

## Características del Panel

El panel utiliza indicadores de color para mostrar el estado del servicio:

## Sistema de Filtros

### Filtros por Período

El panel incluye un sistema de filtros que permite:

- **Períodos predefinidos**: Basados en el día de inicio del período de facturación (8 períodos históricos)
- **Rangos personalizados**: Selección libre de fechas de inicio y fin
- **Validación automática**: Verificación de rangos de fechas válidos
- **Sincronización automática**: Los filtros se sincronizan con los datos de consumo recibidos

### Configuración de Filtros

<Warning>
Los filtros de fecha deben respetar la lógica temporal:
- La fecha de inicio no puede ser posterior a la de fin
- Ambas fechas son requeridas para rangos personalizados
- Los mensajes de error se muestran en tiempo real
</Warning>

```typescript
async applyCustomRange(): Promise<void> {
  this.dateMode = 'custom';
  if (!this.customStart || !this.customEnd) {
    this.filterError = 'Selecciona una fecha de inicio y fin.';
    return;
  }

  if (this.customStart > this.customEnd) {
    this.filterError = 'La fecha de inicio no puede ser posterior a la de fin.';
    return;
  }

  await this.loadConsumption({
    startDate: this.customStart,
    endDate: this.customEnd
  });
}
```

### Períodos Automáticos

Los períodos predefinidos se calculan dinámicamente basándose en:

- **Día de inicio de facturación** configurado en el perfil
- **Fecha de anclaje** del período actual
- **8 períodos históricos** incluyendo el período actual
- **Ajuste automático** para meses con diferentes números de días

```typescript
private translateUnitLabel(label: string, units: number): string {
  const normalized = String(label || '').toLowerCase();
  const translations: Record<string, { singular: string; plural: string }> = {
    minute: { singular: 'minuto', plural: 'minutos' },
    call: { singular: 'llamada', plural: 'llamadas' },
    second: { singular: 'segundo', plural: 'segundos' },
    hour: { singular: 'hora', plural: 'horas' }
  };
  
  const translation = translations[normalized];
  if (translation) {
    return Math.abs(units) === 1 ? translation.singular : translation.plural;
  }
  
  return label;
}
```

## Integración con Permisos

<Info>
El panel solo se renderiza si el usuario tiene permisos para visualizar minutos facturables. Esta verificación se realiza a través del servicio `PermissionsService`.
</Info>

### Control de Visibilidad

```typescript
async ngOnInit(): Promise<void> {
  this.restoreCollapsedState();

  const canView = await this.permissionsService.canViewBillableMinutes();
  if (!canView) {
    return; // Panel no se renderiza
  }

  await this.loadConsumption();
}
```

### Manejo de Estados

El componente maneja tres estados principales:

1. **Sin permisos**: El panel no se renderiza en absoluto
2. **Con permisos, sin datos**: Se muestra el panel con mensaje de error
3. **Con permisos y datos**: Se muestra el panel completo con información

## Persistencia de Estado

### LocalStorage

El panel mantiene su estado de expansión usando localStorage:

- **Clave**: `meetzy_billy_consumption_panel_collapsed`
- **Valor**: String boolean ('true'/'false')
- **Recuperación**: Automática al inicializar el componente
- **Manejo de errores**: Graceful fallback si localStorage no está disponible

```typescript
private restoreCollapsedState(): void {
  try {
    this.collapsed = localStorage.getItem(this.COLLAPSED_KEY) === 'true';
  } catch {
    this.collapsed = false;
  }
}

toggleCollapsed(): void {
  this.collapsed = !this.collapsed;
  try {
    localStorage.setItem(this.COLLAPSED_KEY, String(this.collapsed));
  } catch {
    // Ignore storage failures in private browsing or restricted contexts.
  }
}
```

## Performance y Optimización

### Carga Diferida

<Steps>
  <Step title="Inicialización">
    Solo se cargan los datos básicos de consumo al inicializar
  </Step>
  
  <Step title="Datos Históricos">
    Los gráficos históricos se cargan bajo demanda solo para administradores
  </Step>
  
  <Step title="Chart.js">
    La biblioteca de gráficos se carga únicamente cuando se necesita
  </Step>
</Steps>

### Manejo de Errores

El panel implementa manejo robusto de errores:

- **Errores de API**: Mensajes descriptivos en la interfaz
- **Errores de localStorage**: Fallback silencioso
- **Errores de validación**: Feedback inmediato al usuario

<Warning>
En contextos de navegación privada o con restricciones de almacenamiento, el panel funcionará correctamente pero no recordará el estado de expansión entre sesiones.
</Warning>
