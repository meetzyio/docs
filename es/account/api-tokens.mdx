Los tokens API te permiten acceder programáticamente a la API de Meetzy. Estos tokens funcionan como llaves secretas que autentican tus solicitudes automáticamente.

<Warning>
Los tokens API son credenciales sensibles. Nunca los compartas públicamente ni los expongas en código del lado del cliente. Guárdalos de forma segura como variables de entorno.
</Warning>

## Gestión de Tokens

### Acceder al Gestor de Tokens

1. Navega a tu **Panel de Cuenta**
2. Selecciona **API Tokens** en el menú lateral
3. Verás una lista de todos tus tokens activos

El gestor de tokens muestra:
- **Nombre del token**: Identificador descriptivo
- **Token enmascarado**: Primeros caracteres visibles para identificación
- **Fecha de expiración**: Cuándo expira el token
- **Estado**: Activo o revocado

### Crear un Nuevo Token

<Steps>
<Step title="Abrir modal de creación">
Haz clic en el botón **"New Token"** en la esquina superior derecha del gestor de tokens.
</Step>

<Step title="Configurar el token">
- **Nombre**: Asigna un nombre descriptivo (ej: "API Producción", "Webhook Desarrollo")
- **Expiración**: Selecciona el período de validez:
  - 30 días
  - 90 días  
  - 6 meses
  - 1 año
  - Sin expiración (4 años)
</Step>

<Step title="Validación del nombre">
El nombre del token debe:
- Tener al menos 2 caracteres
- Ser único en tu cuenta
- Ser descriptivo para facilitar la gestión
</Step>

<Step title="Crear y copiar">
1. Haz clic en **"Create Token"**
2. **Copia inmediatamente el token completo**
3. Guárdalo en un lugar seguro

<Warning>
El token completo solo se muestra una vez por seguridad. Si no lo copias ahora, tendrás que crear uno nuevo.
</Warning>
</Step>
</Steps>

## Estado Después de la Creación

Una vez creado exitosamente, verás:

### Modal de Confirmación
- ✅ **Icono de éxito** y nombre del token
- ⚠️ **Advertencia de seguridad** destacada
- 🔑 **Token completo** en formato copiable
- 📋 **Botón "Copy"** que cambia a "Copied!" tras el clic

### Información Importante
- El token se agrega automáticamente a tu lista
- El modal permanece abierto para que copies el valor
- Puedes cerrar el modal haciendo clic fuera o en el botón cerrar

## Revocar Tokens

### Proceso de Revocación

<Steps>
<Step title="Identificar el token">
En la lista de tokens, localiza el que deseas revocar.
</Step>

<Step title="Iniciar revocación">
Haz clic en el **icono de papelera** (🗑️) que aparece al pasar el cursor sobre el token.
</Step>

<Step title="Confirmar acción">
Se abrirá un modal de confirmación mostrando:
- Nombre del token a revocar
- Advertencia sobre la irreversibilidad
- Botones "Cancel" y "Revoke Token"
</Step>

<Step title="Completar revocación">
Haz clic en **"Revoke Token"** para confirmar. El token será eliminado inmediatamente.
</Step>
</Steps>

<Info>
La revocación de tokens es **inmediata e irreversible**. Todas las solicitudes API que usen ese token fallarán instantáneamente.
</Info>

## Conectar por MCP

Además de usarlos directamente contra la API REST, tus tokens API sirven para conectar Meetzy a cualquier cliente compatible con **MCP (Model Context Protocol)**, como **Claude**, **Cursor** o **ChatGPT**. Esto le da al asistente acceso directo a tus llamadas, conversaciones de WhatsApp, evaluaciones y analíticas, todo autenticado con el mismo token que ya generaste arriba.

<Info>
El servidor MCP de Meetzy usa transporte **Streamable HTTP**, por lo que no necesitas instalar nada localmente ni ejecutar un proceso intermedio: basta con apuntar tu cliente a la URL del servidor e incluir tu token como cabecera `Authorization`.
</Info>

### URL del servidor MCP

En la sección **"Conectar por MCP"** del gestor de tokens encontrarás la URL de tu servidor MCP, lista para copiar con el botón dedicado. Junto a ella también verás un botón para copiar un snippet de configuración ya completo con tu token (si acabas de crear uno) o con un marcador `YOUR_API_TOKEN` para que lo reemplaces manualmente.

### Configurar tu cliente

<Tabs>
<Tab title="Cursor">
<Steps>
<Step title="Abrir la configuración de MCP">
En Cursor, ve a **Settings → MCP** y selecciona **"Add new MCP server"**.
</Step>
<Step title="Pegar la configuración">
Añade un bloque como el siguiente, reemplazando `YOUR_API_TOKEN` por el token que copiaste al crearlo:

```json
{
  "mcpServers": {
    "meetzy": {
      "url": "https://api.meetzy.ai/mcp",
      "headers": {
        "Authorization": "Bearer YOUR_API_TOKEN"
      }
    }
  }
}
```
</Step>
<Step title="Guardar y verificar conexión">
Guarda los cambios. Cursor debería mostrar el servidor **meetzy** como conectado, y sus herramientas quedarán disponibles en el chat.
</Step>
</Steps>
</Tab>

<Tab title="Claude">
<Steps>
<Step title="Abrir configuración de conectores">
En Claude Desktop, ve a **Settings → Connectors → Add custom connector**.
</Step>
<Step title="Introducir la URL y el token">
Ingresa la **URL del servidor MCP** (Streamable HTTP) y, en el campo de autenticación, añade la cabecera:

```
Authorization: Bearer YOUR_API_TOKEN
```
</Step>
<Step title="Habilitar el conector">
Activa el conector recién creado. Claude podrá entonces consultar tus llamadas, evaluaciones y analíticas de Meetzy directamente desde la conversación.
</Step>
</Steps>
</Tab>

<Tab title="ChatGPT">
<Steps>
<Step title="Ir a conectores de ChatGPT">
En ChatGPT, ve a **Settings → Connectors → Add connector** (disponible en planes que soportan conectores personalizados).
</Step>
<Step title="Configurar el servidor MCP">
Introduce la **URL del servidor MCP** de Meetzy y configura la autenticación con cabecera personalizada:

```
Authorization: Bearer YOUR_API_TOKEN
```
</Step>
<Step title="Confirmar y usar">
Guarda la conexión. Ya puedes pedirle a ChatGPT que consulte tus llamadas, mensajes de WhatsApp, evaluaciones y métricas de Meetzy usando este conector.
</Step>
</Steps>
</Tab>
</Tabs>

<Warning>
Cualquier cliente MCP que conectes con tu token tendrá el mismo nivel de acceso que ese token te otorga a ti. Usa tokens dedicados y con nombres descriptivos (ej: `mcp-cursor`, `mcp-claude`) para poder revocarlos individualmente sin afectar otras integraciones.
</Warning>

<Info>
**Aislamiento por token**: cada token está vinculado únicamente a los datos de tu cuenta. Un cliente MCP conectado con tu token nunca podrá ver ni acceder a información de otras cuentas o compañías de Meetzy, sin importar qué le preguntes. Si necesitas separar el acceso entre distintos agentes o compañías, crea un token independiente para cada uno.
</Info>

## Mejores Prácticas

<Info>
**Gestión Segura de Tokens:**
- Usa nombres descriptivos que indiquen el propósito
- Establece fechas de expiración apropiadas
- Revoca tokens no utilizados regularmente
- Nunca hardcodees tokens en tu código fuente
</Info>

### Organización Recomendada
- **Producción**: `api-prod-2024` - Sin expiración
- **Desarrollo**: `api-dev-enero` - 90 días  
- **Testing**: `api-test-temp` - 30 días
- **Webhooks**: `webhook-payments` - 1 año
- **MCP**: `mcp-cursor`, `mcp-claude`, `mcp-chatgpt` - según necesidad de cada cliente

### Rotación de Tokens
1. Crea un nuevo token antes de que expire el anterior
2. Actualiza todas las integraciones con el nuevo token
3. Verifica que funcionen correctamente
4. Revoca el token anterior

El sistema te notificará cuando los tokens estén próximos a expirar para facilitar la rotación proactiva.
